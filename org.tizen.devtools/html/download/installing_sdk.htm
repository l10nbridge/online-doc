Installing the Tizen SDK

The following instructions consist of 2 parts: the Tizen SDK installation since version 2.4, and the SDK installation of the earlier versions.

Since version 2.4, Tizen GUI Installer is used instead of the Tizen install manager. Check the SDK version before the installation.

SDK Installation (Since Version 2.4)

You can install the SDK with the Tizen GUI installer or the CLI installer. To update or uninstall the SDK, see Updating the Tizen SDK or Uninstalling the Tizen SDK. For more information about the configuration, see Advanced Configuration.

Before starting the installation, check the prerequisites.

You can install the SDK by using one of the following methods:

Tizen GUI Installer

To install the SDK using the Tizen GUI Installer:

  1. Download the appropriate GUI Installer for your platform.

  2. Run the GUI Installer:

    • On Ubuntu, open the terminal, go to the directory where the installer is downloaded, and enter the chmod +x command to apply the execute permission to the installer file. Execute the installer.

    • On Microsoft Windows®, double-click the installer icon.

    • On Mac OS® X, double-click the .dmg file to mount, and double-click the installer icon.

  3. If you agree to the license terms, select the Accept check box. The Install button is enabled. (To install, you must agree to the license terms.)

  4. Select the SDK installation location.

  5. Click Install.

  6. When the installation is completed, click Done.

Tizen CLI Installer

The CLI (command line interface) provides functional tools for running the CLI Installer without the GUI environment.

To install the SDK using the Tizen CLI Installer:

  1. Download the appropriate CLI Installer for your platform.

  2. Run the CLI Installer:

    • On Microsoft Windows®, the command prompt appears and the installer is automatically executed.

    • On Ubuntu and Mac OS® X, open the terminal, go to the directory where the installer is downloaded, and enter the chmod +x command to apply the execute permission to the installer file.

      To execute the installer, enter the command based on the following syntax:

      tizen-web-cli-<version> [options] [<directory path>]

      Parameters:

      Option Description

      --show-license

      Displays the Tizen SDK software license agreement.

      Use this option alone. Do not use with other options.

      --accept-license

      Accepts the license terms.

      --no-java-check

      Skips the Java version check.
      Argument Description

      directory path

      Specifies the installation directory path.

      If you do not enter the path, the SDK is installed in the default directory (/home/{user}/tizen-sdk).

  3. If you agree to the software license and the license terms, type yes for the conditions.

  4. Enter the SDK installation location.

Starting the Tizen IDE

The Tizen IDE (integrated development environment) is part of the SDK and installed with it. When you start the IDE, you must define the folder for the IDE workspace. If you start with a new version of the IDE, do not use the workspace created for the previous IDE version.

To start the IDE:

  • On Ubuntu®, go to Dashboard Home > Tizen IDE.

  • On Microsoft Windows®, go to Start > All Programs > Tizen SDK > Tizen IDE.

  • On Mac OS® X, go to Launchpad > Tizen IDE.

When you start the IDE for the first time, the IDE displays a welcome page with information on Tizen and on developing with the IDE. On the welcome page, click the Workbench icon to go to the development view.

SDK Installation (Before Version 2.4)

When installing the Tizen SDK, you can select the network installation or SDK image installation. You can also install the SDK using the command line interface. To upgrade the SDK, see Updating the Tizen SDK. For information about SDK extension packages, see Extension SDKs.

Before starting the installation, check the prerequisites.

You can install the Tizen SDK using one of the following methods:

Network Installation

To install the Tizen SDK using the Install Manager:

  1. Download the SDK Install Manager.

  2. Run the Install Manager. The SDK set-up wizard starts.

  3. Click Install.

  4. Select the components you want to install, and click Next.

  5. Accept the terms and conditions, and click Next.

  6. Select the SDK home folder, and click Install.

During the installation, the SDK set-up wizard may require you to enter local administrative privileges. When the installation is complete, the SDK set-up wizard notifies you. This means that the SDK is fully installed and configured on your computer.

SDK Image Installation

To install the SDK using an image file, which is a package containing all the SDK files required for an offline environment:

  1. Download the SDK Install Manager.

  2. Run the SDK Install Manager. The SDK set-up wizard starts.

  3. Click Advanced.

  4. In the Advanced Configuration window, select the SDK Image radio button.

  5. Click the folder button, browse to the SDK image file, and click OK.

  6. Click Install.

  7. Select the components you want to install, and click Next.

  8. Accept the terms and conditions, and click Next.

  9. Select the SDK home folder, and click Install.

When the installation is complete, the SDK set-up wizard notifies you. That means that the SDK is fully installed and configured on your computer.

Command Line Interface Installation

The CLI (command line interface) provides functional tools for running the Install Manager without the GUI environment. It includes the entire Install Manager processes, such as installation, uninstallation, updating, displaying the repository, and the SDK status information.

To use the Tizen CLI:

  1. Download the SDK Install Manager.

  2. If you use Microsoft Windows®, open the command prompt. If you use Ubuntu or Mac OS® X, open the terminal.

  3. Go to the directory where the installer is downloaded.

  4. On Ubuntu and Mac OS® X, enter the chmod +x command to apply the execute permission to the installer file.

  5. Use the following commands:

    • To install the SDK, enter the following command:

      Installation method Description

      Network

      If the SDK is already installed on your computer, the install manager checks whether the installed SDK is the latest.

      • In case the installed SDK is the latest, an error message appears.

      • Otherwise, the install manager updates the installed SDK to the latest.

      Commands:

      Enter –install as an argument for the execution:

      • On Microsoft Windows®, for example:

        > tizen-sdk_2.3.1_rev1_windows-64.exe –install –r http://download.tizen.org/sdk/packages-2.3.1/ -d official –p all –accept_license

      • On Ubuntu and Mac OS® X, for example:

        $ ./tizen-sdk_2.3.1_rev1_ubuntu-64.bin –install –r http://download.tizen.org/sdk/packages-2.3.1/ -d official –p all –accept_license

      Options:

      • -r: Specifies the address of the repository.

      • -d: Specifies the distribution, such as official or develop.

      • -p: Specifies the meta-package type, such as all or typical.

      • -l: Specifies the SDK installation path.

      • -x: Specifies the address of the Proxy server.

      • -accept_license: Required. If you omit this option, the installation process stops.

      SDK Image

      For convenience, place the downloaded SDK image file in the same folder or directory where the install manager is.

      Commands:

      Enter –install as an argument for the execution:

      • On Microsoft Windows®, for example:

        > tizen-sdk_2.3.1_rev1_windows-64.exe  -install –f tizen-sdk –image-TizenSDK_2.3.1_Rev1-windows64.zip –p all –accept_license

      • On Ubuntu and Mac OS® X, for example:

        $ ./tizen-sdk_2.3.1_ rev1_ubuntu-64.bin -install –f tizen-sdk –image-TizenSDK_2.3.1_Rev1-ubuntu64.zip –p all –accept_license

      Options:

      • -f: Required. This option specifies the path of the SDK image file.

      • -p: Specifies the meta-package type, such as all or typical.

      • -l: Specifies the SDK installation path.

      • -accept_license: Required. If you omit this option, the installation process stops.

    • To uninstall the SDK, enter the following command:

      Description

      Commands:

      Enter -remove as an argument for the execution:

      • On Microsoft Windows®, for example:

        > tizen-sdk_2.3.1_rev1_windows-64.exe –remove –p all

      • On Ubuntu and Mac OS® X, for example:

        $ ./tizen-sdk_2.3.1_rev1_ubuntu-64.bin –remove –p all

      Options:

      • -p: Specifies the meta-package type, such as all or typical.

    • To display the information of the installation, enter the following command:

      Description

      Commands:

      Enter –status as an argument for the execution:

      • On Microsoft Windows®, for example:

        > tizen-sdk_2.3.1_rev1_windows-64.exe –status

      • On Ubuntu and Mac OS® X, for example:

        $ ./tizen-sdk_2.3.1_rev1_ubuntu-64.bin –status

    • To display the information of the repository, enter the following command:

      Description

      Commands:

      Enter –repoinfo as an argument for the execution:

      • On Microsoft Windows®, for example:

        > tizen-sdk_2.3.1_rev1_windows-64.exe –repoinfo –r http://download.tizen.org/sdk/packages-2.3.1/

      • On Ubuntu and Mac OS® X, for example:

        $ ./tizen-sdk_2.3.1_rev1_ubuntu-64.bin –repoinfo –r http://download.tizen.org/sdk/packages-2.3.1/

      Options:

      • -r: Specifies the address of the repository.

    • To display the information of the package, enter the following command:

      Description

      Commands:

      Enter –packageinfo as an argument for the execution:

      • On Microsoft Windows®, for example:

        > tizen-sdk_2.3.1_rev1_windows-64.exe –packageinfo –r http://download.tizen.org/sdk/packages-2.3.1/ -d official

      • On Ubuntu and Mac OS® X, for example:

        $ ./tizen-sdk_2.3.1_rev1_ubuntu-64.bin –packageinfo –r http://download.tizen.org/sdk/packages-2.3.1/ -d official

      Options:

      • -r: Specifies the address of the repository.

      • -d: Specifies the distribution, such as official or develop.

Constraints on the CLI Installation

  • When you use –p option, enter the meta-package type. You can check the type by using the -packageinfo command.

  • When you install additional SDK packages, configuration arguments, such as the address of the repository, the distribution, and the installation path, must be the same as when installing the SDK.

  • On Ubuntu and Mac OS® X, if the installation path is under the home directory, the directory path must begin with "/" or "/home", not "home".

  • If you do not enter a target location, the SDK is installed in the default location, such as /home/{user}/tizen-sdk (on Ubuntu and Mac OS® X) or c:\tizen-sdk (on Microsoft Windows®).

  • If you enter the address of a proxy server, you do not need to begin with "http://".

Starting the Tizen IDE

The Tizen IDE (integrated development environment) is part of the SDK and installed with it. When you start the IDE, you must define the folder for the IDE workspace. If you start with a new version of the IDE, do not use the workspace created for the previous IDE version.

To start the IDE:

  • On Ubuntu®, go to Applications > Tizen SDK > Tizen IDE.

  • On Microsoft Windows®, go to Start > Tizen SDK > Tizen IDE.

  • On Mac OS® X, go to the tizen-sdk/ide directory and run the Tizen IDE application.
