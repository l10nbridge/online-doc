Troubleshooting

If you used the Install Manager, troubleshoot your Tizen SDK installation by referring to the following installation log file:

  • On Ubuntu and MacOS®: $HOME/.installmanager/install-log

  • On Windows® 7: %LOCALAPPDATA%\.installmanager\install-log

  • On Windows® XP: %USERPROFILE%\Local Settings\Application Data\.installmanager\install-log

If you used the Update Manager, troubleshoot your Tizen SDK installation by referring to the following installation log file:

  • On Ubuntu and Mac OS®: $HOME/.update-manager/install-log/update-manager

  • On Windows® 8 and 7: %LOCALAPPDATA%\.update-manager\install-log\update-manager

The Install Manager can have compatibility issues in Mac OS® X, even if you have installed a JDK or JRE from the Oracle Website. If the following notification is displayed on the screen, follow the onscreen instructions for Java installation.

Network Connection Failure

If the Install Manager fails to connect to the SDK package server:

  • Check whether you can access the Tizen Web site using the Web browser in the system where you are installing the SDK. If you cannot access the Web site, install the SDK using an SDK image.

  • Check whether you are using a network proxy. If you are, run the Install Manager. Click the Settings button, and select the desired option. If the proxy server requires authentication, an ID and password dialog appears.

    Note
    From Tizen 2.4 official SDK, the Installer and the Update Manager are supported instead of the Install Manager. This troubleshooting method is not applicable since the 2.4 version.

Install Manager Crashes (Ubuntu)

The crashing of the Install Manager as a result of the libdbus-1.so file is a JVM problem in Ubuntu. Try the following:

  1. Install the libdbus-java package:

    sudo apt-get install libdbus-java

  2. Run the Install Manager with the noAutoProxy option:

    ./tizen-sdk-ubuntu.bin -noAutoproxy
Note
If the Install Manager crashes while downloading packages, restart the installation. The Install Manager continues to download the remaining packages.

SDK Image File Dialog Failure (Windows®)

When installing the Tizen SDK using the SDK image, the Cannot open SDK image file dialog box can appear. The dialog box is displayed if you are using JDK 1.6 Update 26. To avoid this problem, ensure that you are using the latest version of Java.

Java Crashes During SDK Installation (Windows® 64-bit)

During the Tizen SDK installation, the Java application can crash unexpectedly, and the following window is displayed.

Figure: Java crash during the SDK installation

To avoid this problem:

  1. Ensure that you are using the latest version of the error DLL file, such as MSVCR100.dll. If the error file version is not the latest, upgrade it.

  2. Update your system with all the Windows-related updates using Windows Update.

Install Manager Does Not Open (Windows®)

If the Install Manager does not open, run the Install Manager with the -help option:

./install-manager.exe -help

If you see the following message, your Java libraries are not suitable to Java executables:

Registry key 'Software\JavaSoft\Java Runtime Environment\CurrentVersion' has value 'XXX', but 'XXX' is required.

The cause is that the Java files in the System32 directory do not match what is listed for the Java Runtime Environment in the Windows registry.

To avoid this problem, copy the java.exe file to the System32 directory.

Emulator Does Not Open (Windows®)

If the Tizen Emulator does not open, you may be facing the same problem as with the Install Manager above. Check the log file at <TIZEN-SDK-DATA>/emulator/vms/<Emulator name>/logs/emulator.log.
