Web Localization

Web Localization is a view-based tool which helps you localize strings, resources, and contents of Tizen web project. The view offers the folder-based localization, by which you can easily localize your project.

Starting Web Localization

Right-click the selected project in the Project Explorer, and then click Localization.

Figure 1. How to open Web Localization

Managing Locale

To localize strings and resource contents, you should select the specific locale(s) in the Overview tab first.

  • In the Overview tab, click the Add locale(s) button, then the New Target Locale dialog appears.

    Figure 2. Web Localization View

  • In the Locale table, choose the locale you want.

    Figure 3. New Target Locale Dialog

  • Overview tab shows selected target locale information.

    Figure 4. Locale information

  • The container named "locales" is created for localized contents. Under the container, a sub-folder for locale is added when you set the locale(for example, "/locales/en-us", "/locales/zn-cn").

Localizing String

In the String tab, you can localize strings and handle them.

  • Click the Add Localization button on the toolbar, then the New Localization dialog appears.

    Figure 5. Add Localization

  • In the dialog, enter the identifier for localized string contents. And enter the value for default and values for locale you specified in the Locale-Value table. You now should be able to click OK to save the localized string contents.

    Figure 6. String Localization

  • You can add the item to the table or remove the item on the table, and change the view criteria of the table between 'ID' and 'Locale'.

    Figure 7. Criteria Change

  • When you have finished the localization of the string, in the application's top-level folder and each locale folder under locales, language.js which contains the localized string contents is created.

    Figure 8. language.js

  • Then <script> element, of which a src attribute is language.js, is added to the index.html.

    • Figure 9. src element in index.html

    Note
    Please see the instruction page about using localized string contents in Guides > Web Application > Tizen Features > Localization

Localizing Resource

With the view, you can localize resource files such as images, sound, video, HTML, and JavaScript more efficiently.

Resource tab displays the resources of the project as a tile form.

Figure 10. Resource tab

If a resource is not localized, the plus button appears on the resource tile. To localize the resource, do the followings:

  • Click the plus button on the resource tile, then Select a file to set dialog appears.
  • In the dialog, select new resource to replace the default resource.

Figure 11. Select new resource

Then the complex folder structures are created and the names of the resource files are replaced.

Folder-based Localization

Localization is the process of adapting an application to suit a specific culture. Localization makes the application access different resources without source code changes. Folder-based localization places the resources inside locale folders of which names are determined by region codes such as "en-us", "ko-kr", and so on. The locale folders are located under the container folder named "locales". The localized resources are used depending on the location settings of the device. If there is no localized resource, the default resource is used.

See the example below. If the locale of a device is "ko-kr", "images/a.png" and "images/c.png" are replaced with "locales/ko-kr/images/a.png" and "locales/ko-kr/images/c.png" each. However "images/b.png" is not replaced because there is no replaceable resource in "locales/ko-kr/images/".

root/
    images/
        a.png
        b.png
        c.png
    locales/
        ko-kr/
            images/
                a.png
                c.png
