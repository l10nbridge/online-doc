UI Builder

The Tizen Web UI Builder is a graphical user interface builder tool that simplifies the creation of Tizen Web UIs by allowing you to arrange widgets using a drag-and-drop WYSIWYG (What You See Is What You Get) editor. The UI Builder supports a variety of UI widgets, and views and editors.

Note
This tool is supported in mobile applications only.

The UI Builder is not only a UI layout code generating tool, it also supports an easy-to-use programming model though the UI Builder project.

Figure: UI Builder

Animator

The Tizen Animator can apply CSS3 animations to Web applications created by the Tizen Web UI Builder. The Animator can operate seamlessly within a UI Builder project. To access it, simply select the Animator tab in the UI Builder editor Page Designer view.

The Animator shares several views with the UI Builder, such as Outline, Properties, and Pages views. The Animator supports a variety of predefined animations using CSS3 properties, such as opacity, translation, and rotation and also creates its own KeyFrame in the Timeline view or the Animator Editor.

Figure: Animator

Configuration Editor

The configuration editor can be used to configure the settings of the UI Builder project. You can define which devices are available in the Web UI Builder and can be selected from the toolbar in the Page Designer editor. Based on the detail information of the selected devices, the UI Builder creates a feature list to be used in the N-Screen view.

To open the configuration editor, double-click the index.config.xml file in the Project Explorer view.

Figure: Configuration editor

The configuration editor consists of the following parts:

  • Device List

    Defines the devices available in the Page Designer editor. You can only use the selected devices in the Page Designer editor.

  • Device Info

    Includes detail information of the device selected in the Device List. For example, the device name, orientation, device-pixel-ratio, device-width, device-height, width, and height.

  • Device Feature

    Shows the feature list for the N-Screen view:

    • Common

      Contains all devices.

    • Orientation

      Contains either portrait or landscape.

    • Pixel-ratio

      Indicates the density of pixels per unit.

    • Resolution

      Indicates the combination of device-width and device-height.

DataBinding View

Data binding is a way to bind data and UI at runtime, so when the data changes, the UI updates automatically.

The DataBinding view helps you to manage the data source and model easily and show the binding information between data model and UI widgets. If the DataBinding view is not visible, open it from Window > Show View > Other > Tizen > DataBinding.

The DataBinding view consists of a data source panel, data model panel, and binding information panel.

Figure: DataBinding view

To bind data between your HTML file in the Page Designer editor and the Data Model panel, you can:

  • Select the data model item in the DataBinding view, and then drag and drop the item to the widget on the Page Designer editor.
  • Use the Set Target dialog to select the HTML element and bindings.

Outline View

The Outline view displays the hierarchical structure of the page currently open in the Page Designer view. If the Outline view is not visible, open it from Window > Show View > Other > General > Outline.

Figure: Outline view

You can manage widgets in the Outline view:

  • Create a widget by dragging it from the Palette to the Outline view.
  • Move a widget by dragging it to a new position.
  • Change the widget ID in the context menu or by pressing the F2 key.
  • Manage selected widgets by cutting, copying, pasting, and deleting them.

Pages View

The Pages view displays a thumbnail image of all the pages in the currently selected project. If the Pages view is not visible, open it from Window > Show View > Other > Tizen > Pages.

Figure: Pages view

You can re-order pages by dragging them to the selected position in the Pages view.

You can create and delete page templates and pages in the Pages view. Most actions are provided through the context menu:

  • To add a basic page:

    Go to Common > Add Basic Page.

  • To add a new page through a page template:
    1. Right-click the view and select New Page.
    2. Select a group.
    3. Select a page template.

      5 page templates are provided by default.

    4. Insert a page ID.
    5. Click Finish.
  • To delete a selected page:

    Right-click the page and select Delete.

  • To create a new page template:
    1. Right-click the view and select Create Template.
    2. Select a group.
    3. Insert a page template name.
    4. Click Finish.
  • To import a page template:
    1. Right-click the view and select Import Template.
    2. Select a page template.
    3. Select a group.
    4. Insert a page template name.
    5. Click Finish.
  • To export a page template:
    1. Right-click the page and select Export Template.
    2. Select a group.
    3. Select a page template.
    4. Select the export location.
    5. Insert the exported template name.

Properties View

The Properties view displays the properties of the widget selected in the Page Designer view. If the Properties view is not visible, open it from Window > Show View > Other > General > Properties.

The Properties helps you to edit widget and CSS properties, and events.

Figure: Properties view

The Properties View consists of Attributes, Style, and Event tabs:

  • Attributes

    In the Attributes tab, you can edit widget attributes. Attributes are categorized into Common and Widget specification. In the Common category, you can edit common widget properties, and in the Widget specification category, you can edit widget-specific properties. The Widget specification category can be divided into categories in accordance with the widget attributes.

    • Common

      The Common category only includes the ID attribute, which is the default widget identifier in the HTML and JavaScript.

    • Widget specification

  • Style

    In the Style tab, you can edit the CSS properties of the selected widget. These properties include, for example, position, size, text, background, and border. If you select a widget in the Page Designer view, the Style tab shows an ID selector of a selected widget. The ID selector is the default CSS selector. You can create or remove the CSS selector of a selected widget.

    • You can edit the CSS properties, such as the left, top, width, and height styles. The Page Designer view reflects the applied style.

    • CSS selector

      To create a CSS selector, click the button, and the CSS Selector dialog opens. The dialog shows the live DOM hierarchy and class of the selected widget, and you can select the selector you want.

      To remove a CSS selector, click the button. If you remove the default CSS selector, the CSS selector itself is not removed, only the CSS style.

  • Action

    In the Action tab, you can create custom actions and create and bind predefined widget actions with JavaScript functions. Under the applicable event section (such as Tap), select the action you want from the Action drop-down menu, and define the additional action details.

    The following actions are available:

    • Run Java Script Function: Create an event handler and bind it with a JavaScript function defined in the Function field. If you click the button next to the Function field, the JavaScript editor opens. When the JavaScript function is focused, you can start writing code.
    • Go to Page: Select the page to move to in the Page field (Next Page, Previous Page, or any of the pages that are already created). You can also select a transition effect.
    • Open URL: When an event occurs, move to the defined URL.
    • Start Animation Group: When an event occurs, start the selected animation group.
    • Pause Animation Group: When an event occurs, pause the selected animation group.
    • Go to Time in Animation Group: When an event occurs, move the selected animation group to the animation time defined in the Time field.

Timeline View Operations

The following operations are possible in the Timeline view:

  • Add an animation group:
    1. Open the animation group drop-down menu.
    2. Select New animation group.
    3. In the New animation group window, define the new animation group name and attributes, and click OK.

      The new animation group is now shown in the outline.

  • Delete an animation group:
    1. Open the animation group drop-down menu.
    2. Select Delete animation group.

      If there is only 1 animation group, it cannot be deleted. If there are 2 or more animation groups, select the ones you want to delete and click OK.

  • Add a predefined KeyFrame unit:
    • Use the toolbar:
      1. Drag the timeline ruler to the point you want.

      2. Click on the toolbar and select emphasis > flash.

        The flash KeyFrame unit is now visible on the timeline.

    • Use the context menu:
      1. Drag the timeline ruler to the point you want.
      2. Right-click the desired line on the timeline and select Add PredefinedKeyFrame > emphasis > flash.

        The flash KeyFrame unit is now visible on the timeline.

  • Add a KeyFrame or a KeyFrame unit:
    1. Drag the timeline ruler to the point you want.

    2. From the selector list in the outline, select the desired selector and click the button.

      The KeyFrame is now visible on the timeline.

    3. Drag the timeline ruler to a new point.
    4. Move the widget in the Animator Editor.

      Or perform the move by entering "100" to the top property field in the Properties view.

      The KeyFrame unit is now visible on the timeline.

  • Delete a KeyFrame or a KeyFrame unit:
    1. Click the timeline ruler, or enter the desired time to Time Edit Box.
    2. Right-click the KeyFrame or KeyFrame unit you want to delete, and select Delete.

  • Preview an animation:
    1. Move the timeline ruler to the desired time and click the button on the toolbar.

      To play an animation group repeatedly, click the button.

    2. To stop the preview, click the button.
  • Copy, cut and paste KeyFrames or KeyFrame units:
    1. After adding it, click the desired KeyFrame or KeyFrame unit.

    2. Right-click the KeyFrame or KeyFrame Unit you want to copy or cut, and select Copy or Cut.

    3. Move the timeline ruler to the wanted time, right-click it and select Paste.
  • Filter animations:

    Click the button on the toolbar to see only animated HTML widgets in the outline animation group.

  • Ease animations:
    1. Click the KeyFrame.
    2. Click the button and select linear.

      The applied linear KeyFrame is now visible in the preview.

  • Create a custom animation:
    1. Add the KeyFrame unit for which you want to create a custom animation.
    2. Right-click the KeyFrame unit, and select Create Custom Animation.

    3. Select the icon, and define the category and custom animation name.

    4. Click Finish.
    5. Add the custom animation like a predefined KeyFrame unit.

  • Delete a custom animation:
    1. Right-click the timeline and select Delete Custom Animation.

    2. Select the custom animations that you want to delete.
    3. Click Finish and confirm the deletion by clicking OK.

      You can check that the selected custom animations have been removed from the predefined list.
