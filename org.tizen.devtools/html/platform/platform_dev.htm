Platform Development

The platform IDE offers a set of tools that helps you develop, for example, preloaded Tizen applications, platform libraries, and device drivers.

Using the Ubuntu Linux version of the Tizen IDE, you can perform, for example, the following tasks:

  • Compile platform modules directly from the Tizen IDE by selecting the appropriate architecture and rootstrap.
  • Browse and clone all existing git repos using the IDE.
  • Update the rootstrap automatically from the IDE.
  • Minimize the terminal window usage by enabling platform development using the IDE.
  • Perform all git operations from the IDE itself (such as add, merge, push, and create branch).
  • Perform module debugging inside the IDE after building and updating the rootstrap.

You can download the latest the Tizen version from https://developer.tizen.org/downloads/tizen-sdk.

Setting up the Platform Development Environment

Note
The Ubuntu 12.04 or higher (32- or 64-bit) operating system version is supported for installing the Tizen SDK for platform development.

To set up the platform development environment, you must install the GBS and MIC development tools and set up the Git, Gerrit and proxy configuration. The following instructions explain the procedure in the Ubuntu 12.04 LTS version. The same instructions apply to other supported operating system versions as well.

To set up the platform development environment:

  1. To install development tools, see Installing Development Tools.

    You need:

    • Git Build System (GBS): command line tool that supports Tizen package development.
    • Image Creator (MIC): command line tool that supports Tizen image creation.
    Note
    Both tools are mandatory for Tizen platform development.
  2. To install the development environment, see Setting up Development Environment.

    These instructions cover:

    • How to register a user account to gain access to tizen.org.
    • How to configure Secure Shell (SSH) for Gerrit access.
    • How to configure Git.
  3. If your system is using a proxy server, see Network Connections for instructions on how to configure the proxy.
