Developing with the Platform IDE

You can use the platform IDE to create projects, build, run, and debug platform modules, and commit and push changes to a repository.

Creating a Platform Project

The Tizen IDE provides functionality for creating projects from templates and samples using the Platform Project Wizard.

To create a project with the Platform Project Wizard:

  1. In the Tizen IDE, select File > New > Project.
  2. In the New Project window, select Tizen > Tizen Platform Project and click Next.

    If the project option you want is not visible, make sure that you are using the correct perspective (Tizen Platform). To switch perspectives, go to Window > Open Perspective > Other and select the perspective you need.

Figure: Platform Project Wizard

Creating a New Platform Project Using Git

The following settings are required before you create a platform project:

  • Set up the platform development environment.
  • In Ubuntu, install the following packages: ruby and expect.
  • If your system is using a proxy server, ensure that the proxy settings are set in the Tizen IDE.

To create a project using Git, see Starting from existing Git Repositories.

Building the Platform Module

You must build the Tizen platform project before you can run and debug the platform.

To build and package the Tizen platform project:

  1. Select a project in the Project Explorer view and select a rootstrap in the Rootstrap View.
  2. In the context menu of selected project, select Build Project.

    The building and packaging progress dialog box is displayed.

    When the packaging is complete, the package file is listed in a sub-folder within the project folder (Debug or Release).

If an error message is displayed in the Console view because of dependency problems, you can install the packages that are needed to build the project using the package manager. Once the installation is complete, build the platform again. You can set GBS options in the IDE for the project.

Package Manager

The package manager is used for managing packages. To run the package manager, right-click the desired rootstrap and click Manage Packages.

You can install packages to the rootstrap or the device by selecting the appropriate tab. On both tabs:

  • Repositories panel:
    • Add: Add a new package repository to the list.
    • Modify: Modify a selected repository.
    • Remove: Remove a selected repository from the list.
    • Move: Move up and down a selected repository on the list.
  • Packages panel:
    • Refresh: Resynchronize the package index files from the package repositories. If repositories are changed, refresh the package index.
    • Upgrade: Perform a distribution upgrade.
    • Install: Install selected packages.
    • Install Local: Install selected package files (.rpm).
    • Uninstall: Uninstall selected packages.
    • Filter: Search a package for categories in All, Installed, and Upgradable.

    To view detailed information about each package, right-click a package name and select Properties.

  • Log panel:
    • Show a log for the packaging command.

Rootstrap View

The Rootstrap View displays available rootstraps and the currently selected rootstrap. You can use the following features of the Rootstrap View:

  • Select a rootstrap.
  • Add or remove a rootstrap.
  • Export a rootstrap.
  • Manage packages.

To set a rootstrap to the project, right-click the desired rootstrap and click Select.

To export a rootstrap as a compressed TAR file, right-click the desired rootstrap and click Export.

Adding a Rootstrap

You can add a rootstrap based on a remote repository.

To add a rootstrap based on a remote repository:

  1. In the Rootstrap View toolbar, select Add Rootstrap.
  2. In the Generate Rootstrap Dialog window, click Add (plus icon).
  3. Enter a value to the Repository URL field and click OK.
  4. Select a value for the Architecture field and click OK.

    If the Generate Immediately check button is selected, the rootstrap is created immediately.

Managing Rootstrap Packages

You can manage rootstrap packages with the package manager. To run the package manager for a rootstrap, right-click the desired rootstrap and click Manage Packages.

GBS Options

The Git Build System (GBS) is used to build a platform project internally. The GBS options can be set in the IDE.

Figure: GBS options

To set the GBS options in the IDE:

  1. Right-click the project in the Project Explorer view.
  2. Select Properties > C/C++ Build > Tizen Settings > GBS Options.
  3. Set the options and click OK.

Running and Debugging the Platform Module

You can run and debug the platform module, and attach a debug process to the project.

Running the Platform Module

To run the platform project:

  1. Connect a device to your computer and select the device in the Connection Explorer view.
  2. In the Project Explorer view, right-click the project and select Run As > Tizen Platform Project.
  3. Choose the install command, such as RPM or pkgcmd.
  4. Click Next in the Launch Configuration dialog.
  5. Enter the command and arguments, and click Finish.

Debugging the Platform Module

You can debug platform projects and also attach a debugging process to the project in the Tizen IDE menu.

To debug the project:

  1. Connect a device to your computer and select the device in the Connection Explorer view.
  2. Set breakpoints in the source code.
  3. In the Project Explorer view, right-click the project and select Debug As > Tizen Platform Project.
  4. Choose the install command, such as RPM or pkgcmd for the device and rootstrap.
  5. Click Next in the Launch Configuration dialog.
  6. Enter the command and arguments, and click Finish.

Attaching a Debug Process

To attach a debug process to the project:

  1. Connect a device to your computer, and select the device in the Connection Explorer view and the running process for attaching.
  2. Set breakpoints in the source code.
  3. In the Project Explorer view, right-click the project and select Debug As > Tizen Platform Project - Attach.
  4. To set the debug launch wizard configuration:
    1. Click Browse, and select the process for attaching.
    2. Click Finish.

    The debugging starts. For more information on debugging, see Debugging Applications.

    After setting the launch configuration, you can debug a project again using the debug toolbar menu and modifying the debug configuration.

Committing Changes and Pushing to a Repository

You can commit source changes and push the changes into the repository using Egit.

Committing Changes

To commit source changes:

  1. Right-click the project and select Team > Commit.
  2. In the Commit Changes dialog, fill in the Commit message and select change files fields.

    To see the changes in a file, double-click it.

  3. Click Commit.

Pushing to a Repository

To push the project to the repository after committing the changes:

  1. Right-click the project and select Team > Push to Upstream.
  2. If you use the Gerrit code review system, modify the ref mapping:
    1. Click Configure.
    2. Select Advanced > Edit(Advanced) in the Ref mapping group.
    3. In the Push Ref Specifications window, select Source ref and modify the values for the Destination ref as refs/for/[branch] format fields.
    4. Click Add Spec, and check the Specifications for push table.
    5. Click Finish.
    6. In the Configure Push window, click Save and Push.
  3. Click OK.
