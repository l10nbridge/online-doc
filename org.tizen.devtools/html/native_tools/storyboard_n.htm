Storyboard

The purpose of the Tizen Storyboard is to provide an interactive view to help in the page transition design. Using the Storyboard, you can visualize page transition in the application.

The StoryBoard is an additional tab in the WYSIWYG Editor view of the UI Builder. The StoryBoard tab shows multiple application pages (views) created with the native UI Builder, and connections between the pages, depicting the transition between them.

Creating a Storyboard Application

To use the Storyboard in your application, you must create a native application project in the IDE using the UI Builder - Single View or UI Builder - Navigation template.

Figure: Creating an application project

Adding Views

When you select the Storyboard tab in the WYSIWYG Editor view, the Storyboard graphical viewer opens. If the application uses the UI Builder - Navigation template, the viewer contains 3 application views with some default connections between them. Otherwise, the viewer contains a single view.

Figure: Views shown in the Storyboard tab

To manage the views:

  • To add a new view, right-click the editor and select Add Empty View.

    Figure: Adding a new view

    Alternatively, add the new view in the Navigation view, and drag and drop the view from the Navigation view to the Storyboard tab.

  • To move the views in the viewer, select the black label containing the view name and drag it to a new position.

Creating Connections

To create connections between 2 views, press Ctrl and click the component on the view from which the transition is to happen, and drag the component to the intended page. When the dragging is finished, a context menu opens, allowing you to select an event from a list of events available for the component.

To create a connection for a UI component:

  1. Select the component from which you want to create connection to some other view in your application.

    Press Ctrl on your keyboard and drag the component to the target view.

  2. When you drop the component on the target view, a list of component events is shown in a context menu. Select one of the events.

    If a connection already exists for an event, it is shown in a disabled state.

    After you select the event, the new connection is displayed in the Storyboard tab and code is generated in the layout.xml file. The event handler code is also generated in a C file.

  3. To view the connection options, right-click the connection:
    • Go to Source Code: Open the C source code of the connection.
    • Delete Connection: Delete the connection.

    If you select Go to Source Code, you can see that 2 callback functions have been automatically generated:

    • One is called before the transition to the target view is performed (in the following image, view1_button1_onclicked).
    • One is called after the transition has been performed (in the following image, view1_button1_onclicked_post).

    You can perform any pre or post transition operations you want within the callback functions.

Changing the Connection Properties

When you select a component or connection in the Storyboard, the editable properties related to it are displayed in the Properties view. For the connection, you can view the transition properties and change the connection target view and title.
