Button

Button is a UI component which works as a clickable input element to trigger events. This document explains the basic usage and main features of button component. For more information, see API Reference.

Basic Usage

Take the following steps to use a button in your application:

  1. Add a button with elm_button_add() function. 
    Evas_Object *button;

button = elm_button_add(parent);
  2. Set a style and fill the layout with a text or an image according to the style.
    • Set a style to the button with elm_object_style_set() function. You can use the default style without this step. 
      elm_object_style_set(button, "circle");
    • Set a label to the button with elm_object_text_set() if the style supports a text part. 
      elm_object_text_set(button, "Home");
    • Set an image object to the button with elm_object_part_content_set() function if the style supports a swallow part. 
      Evas_Object *icon;

elm_object_part_content_set(button, "icon", icon);
  3. Register callback functions. 
    evas_object_smart_callback_add(button, "clicked", clicked_cb, data);

    void clicked_cb(void *data, Evas_Object *obj, void *event_info)
{
   dlog_print(DLOG_INFO, LOG_TAG, "Button clicked\n");
}

The following example shows a simple use case of button component:

Example: A simple use case of button component 
Evas_Object *win, *conf, *nf, *box, *button, *icon;

// Starting right after the basic EFL UI layout code
// (win - conformant - naviframe)

// Add a box to contain a button and push the box into the naviframe
box = elm_box_add(nf);
evas_object_show(box);
elm_naviframe_item_push(nf, "Button", NULL, NULL, box, NULL);

// Add a button and set a style
button = elm_button_add(box);
elm_object_style_set(button, "circle");

// Set a text to the button
elm_object_text_set(button, "Home");

// Add an icon and set it as a content
icon = elm_icon_add(button);
elm_icon_standard_set(icon, "home");
elm_object_part_content_set(button, "icon", icon);

evas_object_show(button);
elm_box_pack_end(box, button);

Features

Autorepeat

Once a button is pressed, a repeated signal is emitted repeatedly until the button is released. We call this feature "autorepeat".

You can disable autorepeat feature:

elm_button_autorepeat_set(button, EINA_FALSE);

You can set the initial timeout before the repeated signal is emitted:

elm_button_autorepeat_initial_timeout_set(button, 5.0);

You can set the interval between 2 repeated signals:

elm_button_autorepeat_gap_timeout_set(button, 0.5);

Callbacks

You can register callback functions connected to the following signals for a button object:

Table: Button callback signals
Signal Description event_info
clicked The button is clicked (press/release). NULL
repeated The button is pressed without releasing it. NULL
pressed The button is pressed. NULL
unpressed The button is released after being pressed. NULL

The signals in API Reference might be more extensive, but only the signals stated above are actually supported by Tizen.
