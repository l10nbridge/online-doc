Datetime

Datetime is a UI component which displays date and time values. This document explains the basic usage and main features of datetime component. For more information, see the Datetime API.

Basic Usage

Take the following steps to use a datetime in your application:

  1. Add a datetime with elm_datetime_add() function. 
    Evas_Object *datetime;

datetime = elm_datetime_add(parent);
  2. Set a style and set a format(The Datetime format is a combination of LIBC standard characters like "%d %b %Y %I : %M %M"). For more information about format characters, see Datetime API.

    • Set a style to the datetime with elm_object_style_set() function. You can use the default style without this step. 
      elm_object_style_set(datetime, "date_layout");
    • Set a format elm_datetime_format_set() 
      elm_datetime_format_set(datetime, "%d%b%Y");
  3. Register callback functions. 
    evas_object_smart_callback_add(datetime, "changed", changed_cb, data);

    void changed_cb(void *data, Evas_Object *obj, void *event_info)
{
   dlog_print(DLOG_INFO, LOG_TAG, "Datetime value changed\n");
}

The following example shows a simple use case of datetime component:

Example: A simple use case of datetime component 
Evas_Object *win, *conf, *nf, *box, *datetime;
time_t local_time;
struct tm *time_info;

// Starting right after the basic EFL UI layout code
// (win - conformant - naviframe)

// Add a box to contain a datetime and push the box into the naviframe
box = elm_box_add(nf);
evas_object_show(box);
elm_naviframe_item_push(nf, "Datetime", NULL, NULL, box, NULL);

// Add a datetime and set a style
datetime = elm_datetime_add(box);
evas_object_size_hint_align_set(datetime, EVAS_HINT_FILL, EVAS_HINT_FILL);
elm_object_style_set(datetime, "date_layout");

// Set a format for datetime
elm_datetime_format_set(datetime, "%d/%b/%Y");

// Get a current local time to set as datetime value
time(NULL);
time_info = localtime(&local_time);
elm_datetime_value_set(datetime, time_info);

evas_object_show(datetime);
elm_box_pack_end(box, datetime);

Configuring Options

Vaule min/max Set

Datetime provides a way to decide range of the value.

Please refer to below range table when you set up time structure.

  • Year : years since 1900. Negative value represents a year below 1900 (year value -30 represents 1870). Year default range is from 70 to 137.
  • Month : default value range is from 0 to 11.
  • Date : default value range is from 1 to 31 according to the month value.
  • Hour : default value is in terms of the 24 hr format (0~23).
  • Miniute : default value range is from 0 to 59.

You can set max time for datetime:

elm_datetime_value_max_set(datetime, tm);

You can set min time for datetime:

elm_datetime_value_min_set(datetime, tm);

Field Value Limit Set

The type of the field.

  • ELM_DATETIME_YEAR : Indicates the Year field
  • ELM_DATETIME_MONTH : Indicates the Month field
  • ELM_DATETIME_DATE : Indicates the Date field
  • ELM_DATETIME_HOUR : Indicates the Hour field
  • ELM_DATETIME_MINUTE : Indicates the Minute field
  • ELM_DATETIME_AMPM : Indicates the AM/PM field

You can set value of range for specific field:

elm_datetime_field_limit_set(datetime, ELM_DATETIME_DATE, 10, 20);

Note
Pay attention to the following UX issue since Tizen 2.3:

The time_layout, time_layout_24hr styles needs a full-length format that includes the year, month, day, hour, minute, and AM/PM. Each style shows specific fields from the format, limited by the UX concept:

  • date_layout (default): Year, month, day
  • time_layout: Hour, minute, AM/PM button
  • time_layout_24hr: Hour, minute

If you call the elm_datetime_field_visible_set() function for a field that is not supported in the current style, the function does not work.

Callbacks

You can register callback functions connected to the following signals for a datetime object:

Table: Datetime callback signals
Signal Description event_info
changed The datetime field values are changed. NULL
language,changed The system locale changes. NULL

The signals in API Reference might be more extensive, but only the signals stated above are actually supported by Tizen.
