Naviframe

Naviframe is a container component that manage many views by stacking them and displaying the top-most one. This document explains the basic usage of naviframe component. For more information, see API Reference.

Basic Usage

Building a layout with a naviframe usually has the following process:

  1. Add a naviframe with elm_naviframe_add() function.
  2. Add an item (a view) and push it to the stack using elm_naviframe_item_push() function. The function takes six parameters and returns a handler of the item. 
    Elm_Object_Item *elm_naviframe_item_push(Evas_Object *obj,
                                         const char *title,
                                         Evas_Object *prev_btn,
                                         Evas_Object *next_btn,
                                         Evas_Object *content,
                                         const char *item_style
                                        );

    In Tizen, "next_btn"(4th parameter) does not be shown, therefore this parameter is recomended to input NULL.
    "prev_btn"(3rd parameter) is supported, but auto pushing is also supported over 2.4 version of Tizen.
    We recommend input NULL to "prev_btn", and then set auto pushed flag (about prev_btn) to true. (likes below) 
    // It makes 'prev_btn' automatically if multiple items are pushed. (default : EINA_FALSE)
elm_naviframe_prev_btn_auto_pushed_set(nf, EINA_TRUE);

    The last parameter is the style of the naviframe item. In Tizen 2.4 mobile, various item styles are provided.

    If an item style supports text parts, fill the text parts with elm_object_item_part_text_set() function.

    elm_object_item_part_text_set(nf_it, "subtitle", "subtitle");

    If an item style supports swallow parts, fill the swallow parts with elm_object_item_part_content_set() function. 
    elm_object_item_part_content_set(nf_it, "title_left_btn", btn);
    The "default" text part is the title, and the "default" swallow part is the main content area. You can fill these parts with elm_naviframe_item_push() function as well.
  3. Repeat step 2. You can pop and delete the top-most view during the process using elm_naviframe_item_pop() function.

The following example shows a typical naviframe in mobile profile:

Example: A simple use case of naviframe component 
Evas_Object *nf, *ly, *tb;
Elm_Object_Item *nf_it;

// Variables for naviframe item push.
const char *title_text = "Naviframe";
Evas_Object *prev_btn = NULL; // recommended NULL, replace to elm_naviframe_prev_btn_auto_pushed_set()
Evas_Object *next_btn = NULL; // recommended NULL
Evas_Object *empty_content = NULL;
const char *item_style_default = NULL;
const char *item_style_tabbar = "tabbar";

// Starting right after the conformant in the basic EFL UI layout code
// (win - conformant)

nf = elm_naviframe_add(ad->conform);
elm_naviframe_prev_btn_auto_pushed_set(nf, EINA_TRUE); // since Tizen 2.4
elm_object_content_set(ad->conform, nf);
evas_object_show(nf);

// Add a layout
ly = elm_layout_add(nf);
elm_layout_theme_set(ly, "layout", "nocontents", "default");
elm_object_part_text_set(ly, "elm.text", "Naviframe Contents Area");

// Add a toolbar which
tb = elm_toolbar_add(nf);
elm_toolbar_shrink_mode_set(tb, ELM_TOOLBAR_SHRINK_EXPAND);
elm_toolbar_transverse_expanded_set(tb, EINA_TRUE);
elm_toolbar_item_append(tb, NULL, "Tab1", NULL, NULL);
elm_toolbar_item_append(tb, NULL, "Tab2", NULL, NULL);
evas_object_show(tb);

// Push a naviframe item (push two items in total to show the back button)
elm_naviframe_item_push(nf, title_text, prev_btn, next_btn, empty_content, item_style_default);

// Push a naviframe item, set the style as "tabbar" and pass the layout as contents
nf_it = elm_naviframe_item_push(nf, title_text, prev_btn, next_btn, ly, item_style_tabbar);

// Set the toolbar into the "tabbar" part of the naviframe item
elm_object_item_part_content_set(nf_it, "tabbar", tb);

Callbacks

You can register callback functions connected to the following signals for a naviframe object:

Table: Naviframe callback signals
Signal Description event_info
transition,finished The transition has finished changing the view. NULL
title,transition,finished The title area transition has finished changing the state of the title. NULL
title,clicked The title area is clicked. NULL

The signals in API Reference might be more extensive, but only the signals stated above are actually supported by Tizen.
