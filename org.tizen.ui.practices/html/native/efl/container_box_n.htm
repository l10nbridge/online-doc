Box

Box is a container component that arranges UI components in a linear order, horizontally or vertically. A box is a non-graphical object: it adds no graphics to or around the objects it holds. This document explains the basic usage of box component. For more information, see API Reference.

Figure: The structure of box component

Basic Usage

Building a layout with a box usually has the following process:

  1. Add a box with elm_box_add() function.
    2. Evas_Object *box;

box = elm_box_add(parent);
  2. Set the direction of the box with elm_box_horizontal_set() function. A box is vertical by default.
    3. elm_box_horizontal_set(box, EINA_TRUE);
  3. Add objects and pack them into the box using elm_box_pack_end(), elm_box_pack_start(), etc. 
    4. Evas_Object *box, btn;

btn = elm_button_add(parent);
elm_box_pack_end(box, btn);

In the following example, five button objects are packed into a vertical box. elm_box_pack_end() function is used to pack each button into the end of the box.

Example: A simple use case of box component 
Evas_Object *win, *conf, *nf, *box, *btn;
char tmp[16];
int i;

// Starting right after the basic EFL UI layout code
// (win - conformant - naviframe)

// Add a box and push it into the naviframe
box = elm_box_add(nf);
evas_object_show(box);
elm_naviframe_item_push(nf, "Box", NULL, NULL, box, NULL);

for(i = 1; i <= 5; i++)
{
   btn = elm_button_add(box);
   snprintf(tmp, sizeof(tmp), "Button %d", i);
   elm_object_text_set(btn, tmp);

   // Set size hints for weight and align
   evas_object_size_hint_weight_set(btn, EVAS_HINT_EXPAND, EVAS_HINT_EXPAND);
   evas_object_size_hint_align_set(btn, EVAS_HINT_FILL, 0.5);

   evas_object_show(btn);
   elm_box_pack_end(box, btn);
}

In the above example, two functions are used to arrange buttons inside the box: evas_object_size_hint_weight_set() and evas_object_size_hint_align_set() function. The former sets a hint on how to resize a given child object within a container's area. The latter sets a hint on how to align a child object inside the boundaries of a container. If you change the weight hint or align hint value, the results will be changed.

Various Box Layouts

When building a box layout, you need to use size hint functions to adjust the size and position of each element inside it. Basically, an object contained in a box take up a space based on its minimum size. If the sum of the objects' minimum size is bigger than the size of the box's parent, the box takes up all the space occupied by the parent, and the size of the parent might be extended to hold the box. If the sum of the objects' minimum size is smaller than the space given for the box, the objects are positioned at the center by default. You can adjust the position of the image objects using elm_box_align_set() function. The function takes two double values, a horizontal value and a vertical value, representing the percentage between 0.0 and 1.0 of the alignment in the horizontal and vertical axes. Remember that a box has a linear structure: the alignment only takes effect in the opposite direction than the one defined with elm_box_horizontal_set() function.

In the following example, two images are packed into a box. Minimum size hints are set to the image objects with elm_object_size_hint_min_set() function. Since the sum of the minimun size is smaller than the box area which is equal to the window area in this case, the images are bound together and positioned at the center.
