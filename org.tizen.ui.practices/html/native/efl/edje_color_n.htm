Using Edje Color Classes

The following example demonstrates how you can change the color of 2 or more parts using Edje color classes.

If a color class is assigned to a part, setting color values to the class causes all assigned parts to have their colors multiplied by the values. Setting values to a color class affects all parts within that color class at process level, while at object level it only affects the parts inside a specified object.

Figure: Edje colorclass application

To create the above simple application, which uses color classes to modify the part colors:

  1. In the Tizen IDE, go to File > New > Tizen Native Project, select a template for creating a UI application with EDC, and create a new project named colorclass.
  2. Open the colorclass.edc file to create the color classes and parts:
    1. Create 3 color classes: 
      color_classes
{
   color_class
   {
      name: "A";
      color: 255 255 0 255;
      color2: 255 0 0 255;
      color3: 0 0 255 255;
   }
   color_class
   {
      name: "B";
      color: 0 255 0 255;
      color2: 0 0 255 255;
      color3: 255 0 0 255;
   }
   color_class
   {
      name: "C";
      color: 125 0 0 255;
      color2: 255 0 0 255;
      color3: 0 0 255 255;
   }
}
    2. Create a collection containing a group for the needed parts, and add the background part: 
      collections
{
   group
   {
      name: "main";
      parts
      {
         /* Background part */
         part
         {
            name: "bg";
            type: RECT;
            description
            {
               state: "default" 0.0;
            }
         }
    3. Add 3 text parts, 6 rectangle parts, and 2 button parts in the group:
      • The text parts are aligned to the left side of the screen, each on its own row.
      • The rows also contain 2 colored rectangles each.

        The color of the rectangles with the color_class A is the same, since they have no base colors. The same applies to the rectangles with the color_class B. However, the rectangles with the color_class C have different base colors set, so they show up with different colors.

      • Assign the color classes to the text and rectangle parts.

        Setting the color class of a part causes its colors to be multiplied by the color values of the color_class.

      To see how the color class affects the shadow and outline colors of the text parts, set those colors (color2 and color3 values) as white (255 255 255 255;) in those parts.

  3. Open the colorclass.c file to create the UI layout:
    1. Define callbacks for button click events.

      The callbacks set the color class, causing the colors of the parts with that color class to be multiplied by the parameter values:

      /* Modify the color class A with button 1 */
static void
btn1_cb(void *data, Evas_Object *obj, void *event_info)
{
    edje_color_class_set("A", rand()%255, rand()%255, rand()%255, 255,
                         rand()%255, rand()%255, rand()%255, 255,
                         rand()%255, rand()%255, rand()%255, 255);
}

/* Modify the color class B with button 2 */
static void
btn2_cb(void *data, Evas_Object *obj, void *event_info)
{
    edje_color_class_set("B", rand()%255, rand()%255, rand()%255, 255,
                         rand()%255, rand()%255, rand()%255, 255,
                         rand()%255, rand()%255, rand()%255, 255);
}
    2. Create the window, conformant, and base layout in the create_base_gui() function: 
      static void
create_base_gui(appdata_s *ad)
{
    char edj_path[PATH_MAX] = {0,};

    /* Window */
    ad->win = elm_win_util_standard_add(PACKAGE, PACKAGE);
    elm_win_conformant_set(ad->win, EINA_TRUE);
    elm_win_autodel_set(ad->win, EINA_TRUE);

    if (elm_win_wm_rotation_supported_get(ad->win)) {
        int rots[4] = {0, 90, 180, 270};
        elm_win_wm_rotation_available_rotations_set(ad->win, (const int *)(&rots), 4);
    }

    evas_object_smart_callback_add(ad->win, "delete,request", win_delete_request_cb, NULL);

    /* Conformant */
    ad->conform = elm_conformant_add(ad->win);
    elm_win_indicator_mode_set(ad->win, ELM_WIN_INDICATOR_SHOW);
    elm_win_indicator_opacity_set(ad->win, ELM_WIN_INDICATOR_OPAQUE);
    evas_object_size_hint_weight_set(ad->conform, EVAS_HINT_EXPAND, EVAS_HINT_EXPAND);
    elm_win_resize_object_add(ad->win, ad->conform);
    evas_object_show(ad->conform);

    /* Base layout */
    app_get_resource(EDJ_FILE, edj_path, (int)PATH_MAX);
    ad->layout = elm_layout_add(ad->win);
    elm_layout_file_set(ad->layout, edj_path, GRP_MAIN);
    evas_object_size_hint_weight_set(ad->layout, EVAS_HINT_EXPAND, EVAS_HINT_EXPAND);
    eext_object_event_callback_add(ad->layout, EEXT_CALLBACK_BACK, layout_back_cb, ad);
    elm_object_content_set(ad->conform, ad->layout);
    3. Add the buttons to the layout and set their callbacks.

      Set the button 1 text to Change A and button 2 text to Change B.

      Clicking Change A changes all colors of all parts with the color_class set as A, but does not affect other parts. The actual color, color2, and color3 values of the affected parts remain unchanged, but they are multiplied by the values from the color class.

          /* Add button 1 */
    ad->btn1 = elm_button_add(ad->win);
    elm_object_part_content_set(ad->layout, "swallow.btn1", ad->btn1);
    evas_object_smart_callback_add(ad->btn1, "clicked", btn1_cb, ad);
    elm_object_part_text_set(ad->btn1, NULL, "Change A");

    /* Add button 2 */
    ad->btn2 = elm_button_add(ad->win);
    elm_object_part_content_set(ad->layout, "swallow.btn2", ad->btn2);
    evas_object_smart_callback_add(ad->btn2, "clicked", btn2_cb, ad);
    elm_object_part_text_set(ad->btn2, NULL, "Change B");

    /* Show the window after the base GUI is set up */
    evas_object_show(ad->win);
}

You can omit the color class declaration in the .edc file and still use it in parts. Then the color classes cause no visual effects on the screen originally, until you change the color class in the code.

For example, if you comment out the color class B in the .edc file, and run the application, the text and rectangular parts with the color class B are invisible at first. Since they have no base color, their shadow and outline color is set to white, and no color class is applied originally to multiply the color values, you cannot see them against the white background. However, when you click Change B, the color class from the callback code is applied, the color of those parts changes, and you can see them.
