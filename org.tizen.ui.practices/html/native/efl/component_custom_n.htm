Customizing Components

Elementary UI components use the EDC themes from the Edje library to manage their look.

Customizing and Expanding Themes

An Elementary theme (consisting of all available styles for the UI components) is an Edje EDC file (.edc) that contains groups composed of part and program blocks. When the application is compiled, the Edje tools compile the EDC file to make an .edj file that is used by the application. For more information on using the EDC language, see Themes.

When the application is run, Elementary must load the styles to display the UI components in the application theme. When looking for the correct style to use, Elementary uses the following process:

  1. Elementary checks the list of overlays, if any are defined.

    An overlay can replace the look of all UI components by overriding the default style of the theme.

  2. If no applicable overlay is found, Elementary checks the theme for the correct style.
  3. If the style is not found in the theme, Elementary checks the extension list.

    An extension, unlike an overlay, provides styles that can be applied to specific UI components only (not all of them).

  4. If the applicable style is not found in the extension list either, Elementary uses the default style of the theme.

To create new themes, overlays, or extensions:

  • Themes

    To create a new theme object, use the elm_theme_new() function. It creates an empty theme that uses the default theme. It has its own private set of extensions and overlays, which are empty by default.

    Specific themes do not fall back to the themes of parent objects. They are not intended for such use.

    Elm_Theme *new_theme = elm_theme_new();

    To apply the new theme to a button UI component and its children:

    // Create a button component 
Evas_Object *button = elm_button_add();

// Set the new theme to the button component 
elm_object_theme_set(button, new_theme);
  • Overlays

    To add an overlay to the application theme, use the elm_theme_overlay_add() function.

    The following example creates an overlay (the theme_button.edj file) with the new default button style that replaces the look of all button components by overriding the default style in the theme:

    elm_theme_overlay_add(NULL, "./theme_button.edj");
    Note
    Be careful when using overlays. For example, in the above example, Elementary uses the new overlay style for all the button components defined in the application. You must make sure that the theme_button.edj file reimplements everything that was previously defined in the theme default style concerning the button component.

    To remove the previously added overlay and return to the default theme:

    elm_theme_overlay_del(NULL, "./theme_button.edj");
  • Extensions

    To create a new extension style and add it to the list of extensions, use the elm_theme_extension_add() function.

    The following example creates a new custom button style defined in the compiled theme_button_style_custom.edj file:

    elm_theme_extension_add(NULL, "./theme_button_style_custom.edj");

    To apply the new custom style to a button component:

    Evas_Object *parent, *button;

// Create a button object 
button = elm_button_add(parent);

// Set its style to "custom" 
elm_object_style_set(button, "custom");
Note
When using the elm_theme_extension_add() or elm_theme_overlay_add() functions to add a new extension or overlay to the default theme, Elementary calls the elm_theme_flush() function. The function flushes the Elementary theme caches, reloading the theme of all UI components that use the default theme.

Using and Customizing Theme Styles

A style is a part of the EDC theme that concerns only 1 UI component. It is defined in the EDC file as a group.

The default theme contains multiple styles for some UI components, such as button. To apply a specific style to a UI component, use the elm_object_style_set() function. To determine the currently used style, use the elm_object_style_get() function.

The following example shows how to set the anchor style for a new button component:

Evas_Object *parent, *button;

// Create a button object 
button = elm_button_add(parent);

// Set its style to "anchor"
elm_object_style_set(button, "anchor");

To modify parts of the default theme without changing the theme itself, you can create new customized styles. For example, you can create a new style for a button component to change its appearance without modifying the default theme.

Note
When creating a new style, you must know how the applicable UI component is themed. A new style (EDC group) always replaces the entire default group used by the UI component. The new style group must contain the same important signals and parts as the default style for the object to behave properly.

For example, to create a new custom style for a check UI component:

  1. Copy the default style of the check UI component from the default theme to a new check_custom.edc EDC file.
  2. In the new EDC file, rename the default style to custom.
  3. Update the style details to fit your needs for the new style.
  4. Store the check_custom.edc file with the new style in the /res/edje/ directory of the application project.
  5. Add the compiled check_custom.edj file to the application as an extension: 
    char edj_path[PATH_MAX] = {0,};

// Get the full path of the edj file 
app_get_resource("/edje/check_custom.edc", edj_path, (int)PATH_MAX);

// Load the new style as an extension 
elm_theme_extension_add(NULL, edj_path);
  6. Apply the new custom style to the applicable check UI components with the elm_object_style_set() function: 
    Evas_Object *parent, *check;

// Create a check object 
check = elm_check_add(parent);

// Set its style to "custom" 
elm_object_style_set(check, "custom");

Example of Customizing the Check Component Style

To create a customized check component style:

  1. Create a new check component style that changes the background and the check images of the UI component to custom images.

    The EDC source file for the check component (the check.edc file) is composed of 2 groups, which define 2 different styles for the check component (the default style and the toggle style):

    group 
{
   name: "elm/check/base/default";
}

group 
{
   name: "elm/check/base/toggle";
}

    Copy the group corresponding to the default style to a new file and rename the group to custom to create a new custom style:

    group 
{
   name: "elm/check/base/custom";
   // Copy of the content of "default" style 
}

  2. In the new elm/check/base/custom group, find the parts you want to modify.

    To modify the background and check images, you must locate the bg and check parts.

    part 
{
   name: "bg";
   mouse_events: 0;
   scale: 1;
   description 
   {
      state: "default" 0.0;
      rel1.offset: 1 1;
      rel2.relative: 0.0 1.0;
      rel2.offset: 1 -2;
      align: 0.0 0.5;
      min: 16 16;
      max: 16 16;
      aspect: 1.0 1.0;
      aspect_preference: VERTICAL;
      image 
      {
         normal: "check_base.png";
         border: 5 5 5 5;
         middle: 0;
      }
      fill.smooth: 0;
   }
}
part 
{
   name: "check";
   mouse_events: 0;
   scale: 1;
   description 
   { 
      state: "default" 0.0;
      rel1 
      {
         to: "bg";
         offset: 1 1;
      }
      rel2 
      {
         to: "bg";
         offset: -2 -2;
      }
      visible: 0;
      color: 255 255 255 255;
      image.normal: "check.png";
   }
   description 
   {
      state: "visible" 0.0;
      inherit: "default" 0.0;
      visible: 1;
   }
   description 
   {
      state: "disabled" 0.0;
      inherit: "default" 0.0;
      visible: 0;
      color: 128 128 128 128;
   }
   description 
   {
      state: "disabled_visible" 0.0;
      inherit: "default" 0.0;
      color: 128 128 128 128;
      visible: 1;
   }
}
    Note
    This example assumes that you are familiar with Edje basics, and does not include all the properties to be modified.

  3. Replace the default images with the new custom images by updating the image.normal properties in both parts:

    • In the background part, replace check_base.png with check_base_custom.png.
    • In the check part, replace check.png with check_custom.png.
    part 
{
   name: "bg";   
   description 
   {
      state: "default" 0.0;
      image 
      {
         normal: "check_base_custom.png";
         border: 5 5 5 5;
         middle: 0;
      }
   }
}
part 
{
   name: "check";      
   description 
   {
      state: "default" 0.0;           
      image.normal: "check_custom.png";
   }   
}
    Note
    This example assumes that the custom images are the same size as the default images.

Some parts in the EDC file can interact with Elementary:

  • You can modify the content of the TEXT type parts using the elm_object_part_set_text() function.

    For example, the check component default style has a TEXT type part called elm.text:

    part 
{
   name: "elm.text";
   type: TEXT;
   mouse_events: 0;
   scale: 1;
   description
   {
      state: "default" 0.0;
      visible: 0;
      rel1.to_x: "elm.swallow.content";
      rel1.relative: 1.0 0.0;
      rel1.offset: 1 1;
      rel2.relative: 1.0 1.0;
      rel2.offset: -2 -2;
      color: 0 0 0 255;
      text 
      {
         font: "Sans,Edje-Vera";
         size: 10;
         min: 0 1;
         align: -1.0 0.5;
      }
   }
   description
   {
      state: "visible" 0.0;
      inherit: "default" 0.0;
      visible: 1;
      text.min: 1 1;
   }
   description 
   {
      state: "disabled" 0.0;
      inherit: "default" 0.0;
      color: 0 0 0 128;
      color3: 0 0 0 0;
   }
   description
   {
      state: "disabled_visible" 0.0;
      inherit: "default" 0.0;
      color: 0 0 0 128;
      color3: 0 0 0 0;
      visible: 1;
      text.min: 1 1;
   }
}

    You can modify the content of the part from the application. The EDC theme controls the position of the text, its size, color, and look and feel.

    Evas_Object *parent, *check2;

// Create a check object 
check2 = elm_check_add(parent);

// Set the text of the check object 
elm_object_part_text_set(check2, "elm.text", "Test text");
  • You can modify the content of the SWALLOW type parts using the elm_object_part_content_set() function.

    For example, the check component default style has a SWALLOW type part called elm.swallow.content:

    part 
{
   name: "elm.swallow.content";
   type: SWALLOW;
   scale: 1;
   description 
   {
      state: "default" 0.0;
      fixed: 1 0;
      visible: 0;
      align: 0.0 0.5;
      rel1.to_x: "bg";
      rel1.relative: 1.0 0.0;
      rel1.offset: 1 1;
      rel2.to_x: "bg";
      rel2.offset: 1 -2;
      rel2.relative: 1.0 1.0;
   }
   description
   {
      state: "visible" 0.0;
      inherit: "default" 0.0;
      fixed: 1 0;
      visible: 1;
      aspect: 1.0 1.0;
      min: 16 16;
   }
   description 
   {
      state: "disabled" 0.0;
      inherit: "default" 0.0;
      color: 255 255 255 128;
   }
   description 
   {
      state: "disabled_visible" 0.0;
      inherit: "default" 0.0;
      color: 255 255 255 128;
      fixed: 1 0;
      visible: 1;
      aspect: 1.0 1.0;
   }
}

    You can push content (any kind of Evas_Object) to the part from the application. The EDC theme controls the size and position of the pushed content.

    Evas_Object *parent, *check1, *content;

// Create a check object 
check1 = elm_check_add(parent);

// Set the content of the check object 
elm_object_part_content_set(check1, "elm.swallow.content", content);
    Note
    If you want to control more content from the application, you can add new SWALLOW parts when customizing a UI component style. However, remember that removing existing SWALLOW parts changes the UI component behavior.

For more information on customizing the look and feel of the UI components with custom styles, see the ThemeExtension sample.

Example of Customizing the Genlist Component Item Style

The genlist component has item styles as well as component styles to enable each item to have various layouts. To customize the item styles:

  1. Name the genlist item group appropriately in the new EDC file.

    The genlist item group name in the compiled EDJ file is based on the elm/genlist/item/<item_style>/<genlist_style> structure. For example:

    • If the "default" style genlist has the "custom" item style, the item group name is:

      elm/genlist/item/custom/default

    • If the "effect" style genlist has the "custom" item style, the item group name is:

      elm/genlist/item/custom/effect

    To use a specific genlist and item style, set them as in the following example. The following example loads the elm/genlist/item/custom/effect group (the "effect" genlist style and the "custom" item style) for the genlist item. If the "custom" item style for "effect" is missing, it searches for elm/genlist/item/custom/default.

    Evas_Object *genlist = elm_genlist_add(parent);
elm_object_style_set(genlist, "effect");

Elm_Genlist_Item_Class *itc = elm_genlist_item_class_new();
itc->item_style = "custom";
elm_genlist_item_append(genlist, itc, NULL, NULL, ELM_GENLIST_ITEM_NONE, NULL, NULL);
  2. Add the EDJ file with the customized genlist item styles to the theme.

    Before using them, append the customized item styles into the theme hierarchy with the elm_theme_overlay_add() or elm_theme_extension_add() function.

    If you want to overwrite the existing genlist item style, use overlay or extension.

    elm_theme_extension_add(NULL, "/full/path/of/genlist_item_style_customization.edj");
  3. Customize the genlist item styles.

    Customizing the genlist item styles is similar to customizing a normal layout. Usually, the item group consists of several SWALLOW and TEXT (or TEXTBLOCK) parts.

    The following example shows a simple genlist item style that has a top TEXT part and a bottom SWALLOW part.

    collections 
{
   group 
   { 
      name: "elm/genlist/item/custom/default";
      alias: "elm/genlist/item_compress/custom/default"; // Set alias for the ELM_LIST_COMPRESS mode

      // data.item: "banded_bg_area" "elm.swallow.bg"; // Set the background area for the banded UX
      data.item: "texts" "elm.text"; // List text parts
      data.item: "contents" "elm.swallow.content"; // List swallow parts

      parts 
      {
         // Add banded_bg_area swallow to use the banded UX
         // part 
         // { 
            // name: "elm.swallow.bg";
            // type: SWALLOW;
            // scale: 1;
            // description 
            // { 
               // state: "default" 0.0;
            // }
         // }

         // Background part determines the pressed effect
         // See the "programs" block at the end of this example
         part 
         { 
            name: "bg";
            type: RECT;
            scale: 1;
            description 
            { 
               state: "default" 0.0;
               color: 128 128 128 255;
            }
            description 
            { 
               state: "pressed" 0.0;
               inherit: "default" 0.0;
               color: 0 0 0 255;
            }
         }

         // top_padding expands from the top edge of the item
         part 
         { 
            name: "top_padding";
            type: SPACER;
            scale: 1;
            description 
            { 
               state: "default" 0.0;
               rel1.relative: 0.0 0.0; // Top-left point
               rel2.relative: 1.0 0.0; // Bottom-right point
               align: 0.5 0.0; // Align shows to what direction this part expands
               min: 0 10; // Minimum size of this part
               fixed: 0 1; // Height is fixed to  make additional calculation unnecessary
            }
         }

         // left_padding expands from the left edge of the item
         part 
         { 
            name: "left_padding";
            type: SPACER;
            scale: 1;
            description 
            { 
               state: "default" 0.0;
               rel1.relative: 0.0 0.0;
               rel2.relative: 0.0 1.0;
               align: 0.0 0.5;
               min: 10 0;
               fixed: 1 0;
            }
         }

         // right_padding expands from the right edge of the item
         part 
         { 
            name: "right_padding";
            type: SPACER;
            scale: 1;
            description 
            { 
               state: "default" 0.0;
               rel1.relative: 1.0 0.0;
               rel2.relative: 1.0 1.0;
               align: 1.0 0.5;
               min: 10 0;
               fixed: 1 0;
            }
         }

         // bottom_padding expands from the bottom edge of the item
         part 
         { 
            name: "bottom_padding";
            type: SPACER;
            scale: 1;
            description 
            { 
               state: "default" 0.0;
               rel1.relative: 0.0 1.0;
               rel2.relative: 1.0 1.0;
               align: 0.5 1.0;
               min: 0 10;
               fixed: 0 1;
            }
         }

         // Separator line is used between items; it overlaps with the bottom_padding part
         part 
         { 
            name: "bottom_line";
            type: RECT;
            scale: 1;
            description 
            { 
               state: "default" 0.0;
               rel1.relative: 0.0 1.0;
               rel2.relative: 1.0 1.0;
               align: 0.5 1.0;
               min: 0 2;
               fixed: 0 1;
               color: 64 64 64 255;
            }
         }

         // elm.text lies between top_padding and mid_padding vertically,
         // and between left_padding and right_padding horizontally
         part 
         { 
            name: "elm.text";
            type: TEXT;
            scale: 1;
            description 
            { 
               state: "default" 0.0;
               rel1 
               { 
                  to_x: "left_padding";
                  to_y: "top_padding"; 
                  relative: 1.0 1.0; 
               }
               rel2 
               { 
                  to_x: "right_padding"; 
                  to_y: "mid_padding";
                  relative: 0.0 0.0; 
               }
               align: 0.5 0.5;
               text 
               {
                  font: "DejaVu Sans";
                  size: 12;
                  min: 0 1;
                  align: 0.5 0.5;
               }
            }
         }

         // mid_padding expands upside from the top edge of elm.swallow.content
         // but lies between left_padding and right_padding
         part 
         { 
            name: "mid_padding";
            type: SPACER;
            scale: 1;
            description 
            { 
               state: "default" 0.0;
               rel1 
               { 
                  to_x: "left_padding"; 
                  to_y: "elm.swallow.content"; 
                  relative: 1.0 0.0; 
               }
               rel2 
               { 
                  to_x: "right_padding"; 
                  to_y: "elm.swallow.content"; 
                  relative: 0.0 0.0; 
               }
               align: 0.5 1.0;
               min: 0 5;
               fixed: 0 1;
            }
         }

         // elm.swallow.content expands upside from the top edge of bottom_padding
         // but lies between left_padding and right_padding
         part 
         { 
            name: "elm.swallow.content";
            type: SWALLOW;
            scale: 1;
            description 
            { 
               state: "default" 0.0;
               rel1 
               { 
                  to_x: "left_padding"; 
                  to_y: "bottom_padding"; 
                  relative: 1.0 0.0;
               }
               rel2 
               { 
                  to_x: "right_padding"; 
                  to_y: "bottom_padding"; 
                  relative: 0.0 0.0; 
               }
               align: 0.5 1.0;
               fixed: 0 1;
            }
         }
      }
      programs 
      {
         program 
         {
            signal: "elm,state,selected";
            source: "elm";
            action: STATE_SET "pressed" 0.0;
            target: "bg";
         }
         program 
         {
            signal: "elm,state,unselected";
            source: "elm";
            action: STATE_SET "default" 0.0;
            target: "bg";
         }
      }
   }
}
