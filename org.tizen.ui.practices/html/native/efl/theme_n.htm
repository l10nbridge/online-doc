Themes: Specifying the UI Look and Feel

Tizen provides powerful theme features to easily change your application UI's look and feel.

A theme is a collection of visual identities, which provide the graphical appearance of applications. A theme comprises a layout, shapes, fonts, and colors for the UI components. The Tizen platform provides a default theme for UI components based on the Tizen UX design concept (AIR â€“ light and simple). To change the current look and feel, you can replace the default theme with a customized theme package. EFL can even change the application theme at runtime without restarting the application.

Elementary UI components use the Edje library, which provides themes. With Edje, you can customize the application theme and individual UI components, and create your own Edje objects to modify the application appearance with a unique UI. Each application can have its own customized theme to overlay the default theme.

Edje supports separation between layout and logic. While it is possible to implement both UI and logic in the C code, a loose coupling is more convenient when you want to revise, add, or delete UI components. In EFL, the UI is handled by Edje in EDC files, while the application logic is handled in the C code:

  • Edje is designed as a theme layer for moving the look, feel, and animation of objects into separate data files loaded at runtime. Using a plain text EDC file, you can create your application's layout without writing a single line of C code. For the basics of writing and compiling an EDC file, see Introduction to EDC Programming.
  • The EDC file can be used to position and resize UI components to specific positions, relative to another component on the screen. For more information, see Positioning Parts with the EDC File.
  • The EDC file consists of blocks, properties, and attributes, implemented in a specific hierarchy and structure. For the details of the EDC file content, see EDJE Data Collection Reference.
  • You can optimize EDC to keep the EDC files as short and simple as possible, while ensuring optimal performance and behavior. For more information, see Optimizing EDC.
