File Transfer

This tutorial demonstrates how to download files from remote servers and to upload files to them. It shows how to abort a transfer and how to track its progress too.

Prerequisites

To download files, the application has to request permission by adding the corresponding privileges to the config.xml file.

<tizen:privilege name="http://tizen.org/privilege/filesystem.read"/>
<tizen:privilege name="http://tizen.org/privilege/filesystem.write"/>

To upload files, the application has to request permission by adding the corresponding privileges to the config.xml file.

<tizen:privilege name="http://tizen.org/privilege/filesystem.read"/>
<tizen:privilege name="http://tizen.org/privilege/internet"/>

To abort a transfer, the application has to request permission by adding the corresponding privileges to the config.xml file.

<tizen:privilege name="http://tizen.org/privilege/internet"/>

Download

The following example demonstrates how to download a file from a remote server. 

 var destinationURL; // should be a valid url eg. cdvfile://localhost/persistent/path/to/file.txt
                     // although the file doesn't need to exist before the operation

 var fileTransfer = new FileTransfer();
 var uri = encodeURI('http://some.server.com/download.php');

 fileTransfer.download(
   uri,
   destinationURL,
   function(entry) {
     console.log('download complete: ' + entry.toURL());
   },
   function(error) {
     console.log('download error source ' + error.source);
     console.log('download error target ' + error.target);
     console.log('upload error code' + error.code);
   },
   false,
   {
     headers: {
       'Authorization': 'Basic dGVzdHVzZXJuYW1lOnRlc3RwYXNzd29yZA=='
     }
   }
 );

The output: 

download complete: file:///home/owner/apps_rw/Gk6hf8hjk/tmp/file.txt

Upload

The following example demonstrates how to upload a file to a remote server: 

 var fileURL; // should be a valid url eg. cdvfile://localhost/persistent/path/to/file.txt
              // the file should exist before the operation

 var win = function (r) {
   console.log('Code = ' + r.responseCode);
   console.log('Response = ' + r.response);
   console.log('Sent = ' + r.bytesSent);
 }

 var fail = function (error) {
   alert('An error has occurred: Code = ' + error.code);
   console.log('upload error source ' + error.source);
   console.log('upload error target ' + error.target);
 }

 var ft = new FileTransfer();
 ft.upload(fileURL, encodeURI('http://some.server.com/upload.php'), win, fail);

The output: 

Code = 200
Response = OK
Sent = 1024

Abort

The following example shows how to terminate a transfer: 

var fileURL; // should be a valid url eg. cdvfile://localhost/persistent/path/to/file.txt

 var win = function(r) {
   console.log('Should not be called.');
 }

 var fail = function(error) {
   // error.code == FileTransferError.ABORT_ERR
   alert('An error has occurred: Code = ' + error.code);
   console.log('upload error source ' + error.source);
   console.log('upload error target ' + error.target);
 };

 var ft = new FileTransfer();
 ft.upload(fileURL, encodeURI('http://some.server.com/upload.php'), win, fail);
 ft.abort();

The output: 

An error has occured: Code = 4
upload error source file:///home/owner/apps_rw/Gk6hf8hjk/tmp/file.txt
upload error target http://some.server.com/file.txt

Tracking Progress

The following example demontrates how to set up a tracking function and how tracsk a progress of a transfer: 

var fileURL; // should be a valid url eg. cdvfile://localhost/persistent/path/to/file.txt

 var win = function(r) {
   console.log('Success.  File uploaded.');
 }

 var fail = function(error) {
   // error.code == FileTransferError.ABORT_ERR
   alert('An error has occurred: Code = ' + error.code);
   console.log('upload error source ' + error.source);
   console.log('upload error target ' + error.target);
 };

 var ft = new FileTransfer();

 ft.onprogress = function(event) {
   console.log('uploaded: ' + event.loaded);
 };

 ft.upload(fileURL, encodeURI('http://some.server.com/upload.php'), win, fail);

The output: 

uploaded: 512
uploaded: 1024
Success.  File uploaded.
