Sync Adapter: Implementing the Sync Mechanism

This tutorial demonstrates how you can set callbacks to start or cancel a sync operation.

Note
This feature is supported in mobile applications only.

Warm-up

Become familiar with the Sync Adapter API basics by learning about:

  • Setting the Callback Functions

    Set the client callback functions for starting or canceling sync operations.

Setting the Callback Functions

To set callbacks to receive notifications about sync operations:

  1. To use the features of the Sync Adapter API, include the <sync_adapter.h> header file in your application: 
    #include <sync_adapter.h>
  2. Initialize the sync adapter instance using the sync_adapter_init() function: 
    int result;
const char* capability = "http://tizen.org/account/capability/calendar";
result = sync_adapter_init(capability);

    If no capabilities are defined (you do not want to receive notifications about calendar or contact data changes on the device):

    result = sync_adapter_init(NULL);
  3. Subscribe to the callback functions to receive notifications for the sync operation when a specific event or condition is detected on the device: 
    result = sync_adapter_set_callbacks(on_start_cb, on_cancel_cb);

    When a specific event is detected, the sync_adapter_start_sync_cb or sync_adapter_cancel_sync_cb callback is invoked.

  4. When the on_start_cb() callback is invoked, the predefined sync process is performed inside the callback function. The on_cancel_cb() callback works in a similar way and cancels the sync operation. 
    int on_start_cb(account_h account, bundle *extras, const char *capability)
{
   // Start the sync operation
}

int on_cancel_cb(account_h account, const char *capability)
{
   // Cancel the sync operation
}
  5. When the sync operation is no longer needed, destroy the sync adapter instance: 
    result = sync_adapter_unset_callbacks();
sync_adapter_destroy();
