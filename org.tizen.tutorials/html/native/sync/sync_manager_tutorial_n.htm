Sync Manager: Managing Sync-related Activities

This tutorial demonstrates how you can manage the data synchronizing schedule and control the sync operations manually or periodically.

Note
This feature is supported in mobile applications only.

Warm-up

Become familiar with the Sync Manager API basics by learning about:

  • Initializing the On-demand Sync

    Add an on-demand sync job scheduling for a one-time operation.

  • Initializing the Periodic Sync

    Add a periodic sync job scheduling with a recurring cycle.

Initializing the On-demand Sync

To inform the service application of a one-time sync operation:

  1. To use the features of the Sync Manager API, include the <sync_manager.h> header file in your application: 
    #include <sync_manager.h>
  2. Set the sync adapter callbacks: 
    #include <sync_adapter.h>

const char* capability = "http://tizen.org/account/capability/contact";
int result;
result = sync_adapter_init(capability);
result = sync_adapter_set_callbacks(on_start_cb, on_cancel_cb);

    If no capabilities are defined (you do not want to receive notifications about calendar or contact data changes on the device):

    result = sync_adapter_init(NULL);
result = sync_adapter_set_callbacks(on_start_cb, on_cancel_cb);
  3. Connect to the sync manager service using the sync_manager_connect() function: 
    result = sync_manager_connect();
  4. If you want to use an account (track calendar or contact data changes on the device), create the account and the bundle to obtain the parameters for the sync_manager_add_sync_job() function you need later on.

    The account handle is not a mandatory parameter, because it is meant for an application that uses the sync manager without an account. For more information, see Creating and Managing an Account and Managing and Using the Bundle Content.

    account_h account = NULL;
int account_id = -1;
bundle *extra = NULL;

account_connect();
account_create(&account);
account_set_user_name(account, "test_name");
account_set_email_address(account, "test_email@samsung.com");
account_set_package_name(account, "data-sync-module");
account_set_sync_support(account, ACCOUNT_SUPPORTS_SYNC);
account_insert_to_db(account, &account_id);
account_disconnect();

extra = bundle_create();
bundle_add(extra, SYNC_OPTION_NO_RETRY, "true");
bundle_add(extra, SYNC_OPTION_EXPEDITED, "false");

    This example creates a bundle and adds options for sync-related information. Since the SYNC_OPTION_NO_RETRY is set to true, the sync service does not retry the operation if the sync operation fails.

  5. To add an on-demand sync job: 
    result = sync_manager_add_sync_job(account, capability, extra);

    If no account is used:

    result = sync_manager_add_sync_job(NULL, NULL, extra);

    If the sync job addition process succeeds, the SYNC_ERROR_NONE value is returned.

  6. When the on-demand sync is no longer needed, remove it with the sync_manager_remove_sync_job() function. If you want to stop using the account too, clean up the account handle.

    At the end, disconnect from the sync manager and release the resources with the sync_manager_disconnect() function.

    result = sync_manager_remove_sync_job(account, capability);

account_connect();
account_delete_from_db_by_package_name("data-sync-module");
account_destroy(account);
account_disconnect();

sync_manager_disconnect();
sync_adapter_unset_callbacks();
sync_adapter_destroy();

    If no account is used:

    result = sync_manager_remove_sync_job(NULL, NULL);

sync_manager_disconnect();
sync_adapter_unset_callbacks();
sync_adapter_destroy();

Initializing the Periodic Sync

To inform the service application of a periodical sync operation and its sync interval:

  1. To use the features of the Sync Manager API, include the <sync_manager.h> header file in your application: 
    #include <sync_manager.h>
  2. Set the sync adapter callbacks: 
    #include <sync_adapter.h>

const char* capability = "http://tizen.org/account/capability/contact";
int result;
result = sync_adapter_init(capability);
result = sync_adapter_set_callbacks(on_start_cb, on_cancel_cb);

    If no capabilities are defined (you do not want to receive notifications about calendar or contact data changes on the device):

    result = sync_adapter_init(NULL);
result = sync_adapter_set_callbacks(on_start_cb, on_cancel_cb);
  3. Connect to the sync manager service using the sync_manager_connect() function: 
    result = sync_manager_connect();
  4. If you want to use an account (track calendar or contact data changes on the device), create the account and the bundle to obtain the parameters for the sync_manager_add_sync_job() function you need later on.

    The account handle is not a mandatory parameter, because it is meant for an application that uses the sync manager without an account. For more information, see Creating and Managing an Account and Managing and Using the Bundle Content.

    bundle *extra = NULL;
extra = bundle_create();
bundle_add(extra, SYNC_OPTION_NO_RETRY, "false");
bundle_add(extra, SYNC_OPTION_EXPEDITED, "false");

    This example creates a bundle and adds options for sync-related information. All the sync options are set to false for a periodic sync request. The sync service tries again when the sync operation has failed and does not grant priority for the sync operation.

  5. To add a periodic sync job: 
    sync_period_e sync_period = SYNC_PERIOD_INTERVAL_30MIN;
result = sync_manager_add_periodic_sync_job(account, capability, extra, sync_period);

    If no account is used:

    result = sync_manager_add_periodic_sync_job(NULL, NULL, extra, sync_period);

    If the sync job addition process succeeds, the SYNC_ERROR_NONE value is returned. The last parameter stands for the time interval of the periodic sync.

  6. When the periodic sync is no longer needed, remove it with the sync_manager_remove_periodic_sync_job() function. If you want to stop using the account too, clean up the account handle, as shown in the Initializing the On-demand Sync use case.

    At the end, disconnect from the sync manager and release the resources with the sync_manager_disconnect() function.

    result = sync_manager_remove_periodic_sync_job(account, capability);

sync_manager_disconnect();
sync_adapter_unset_callbacks();
sync_adapter_destroy();

    If no account is used:

    result = sync_manager_remove_periodic_sync_job(NULL, NULL);

sync_manager_disconnect();
sync_adapter_unset_callbacks();
sync_adapter_destroy();
