Sample Account Overview

The Sample Account application demonstrates how to implement a Account provider which can add and configure account.

This application uses Account Manager API to control account.

Figure: Sample Account view

The application is responsible for displaying account signin screen and configure screen.

The application main window consists of 2 areas:

  • Singin view : displays account signin view and handel adding account
  • Configure View : displays account information and setting for selected account

Prerequisites

To ensure proper application execution, following privileges must be set:

  • http://tizen.org/privilege/account.read
  • http://tizen.org/privilege/account.write

Implementation

To run this application, go Settings > Accounts. Then you can see 'SampleAccount' on Add account list.

Adding Account in Signin View

If singin button in signin view pressed, this application add account with inputted text.

The account_create() function is used to initialize the account manager handle, which is later used to set account data.

int ret = 0;
account_h account = NULL;

/* create account handle */
ret = account_create(&account);

if(ret != ACCOUNT_ERROR_NONE)
{
   // Error handling
   return;
}

After set user ID with account_set_user_name() function and set PASS with account_set_access_token() function.

You can add various other account data with functions which start with account_set. Please check Account Manager.

const char *user_name = NULL, *password = NULL;

/* set account data */
ret = account_set_user_name(account, user_name);

if(ret != ACCOUNT_ERROR_NONE)
{
   // Error handling
   return;
}

ret = account_set_access_token(account, password);

if(ret != ACCOUNT_ERROR_NONE)
{
   // Error handling
   return;

Once acocunt data set, need to insert data to account db. Account Manager provides account_insert_to_db() function.

account_insert_to_db() function hand inserted data's account id as an 2nd parameter.

int account_id = 0;

/* insert data to DB */
ret = account_insert_to_db(account, &account_id);

if(ret != ACCOUNT_ERROR_NONE)
{
   // Error handling
   return;
}

Then release account handle with account_destroy() function.

/* release account handle */
ret = account_destroy(account);

if(ret != ACCOUNT_ERROR_NONE)
{
   // Error handling
   return;
}

Getting Account in Configure View

When you select account which you added in the Account list, account id would hand to this application.

Then get account data with account_query_account_by_account_id() function using handed account id.

If you add more account data during adding account, you could get others with functions start with account_get.

In this application, just get user ID with account_get_user_name() function.

int account_id = handed_value;
account_h account = NULL;
int ret = 0;

/* account_create() needed, please refer Adding Account part */

/* get account data with account_id */
ret = account_query_account_by_account_id(account_id, &account);

if(ret != ACCOUNT_ERROR_NONE)
{
   // Error handling
}

/* get user ID from account data */
ret = account_get_user_name(account, &account_user_id);

if (ret != ACCOUNT_ERROR_NONE) 
{
   // Error handling
}

/* account_destroy() needed, please refer Adding Account part */

Deleting Account in Configure View

If singout button in configure view pressed, this application remove selected account from account DB.

To delete account, use account_get_user_name() with handed account id.

There are several functions provided for deleting account. Please refer Account Manager functions which start with account_delete.

int account_id = handed_value;
int ret = 0;

/* signout current account */
ret = account_delete_from_db_by_id(account_id);

if (ret != ACCOUNT_ERROR_NONE) 
{
   // Error handling
}
