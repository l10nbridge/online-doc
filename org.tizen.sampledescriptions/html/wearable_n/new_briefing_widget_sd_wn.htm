New Briefing Widget Sample Overview

The New Briefing widget sample demonstrates how to manage text objects and image objects to create a complete widget view.

The following figure illustrates the main screen of the New Briefing widget.

Figure: New Briefing main view

This sample consists of the view and main operation parts.

View

The view_create() function creates the widget view frame that consists of the window, text parts, and image parts.

view_create(context, w, h);

The following figure illustrates the view frame.

Figure: New Briefing view frame

To create the parts:

  • Text

    You can fill out the text parts with the part name and the text using the view_set_text() function.

    Implement the data_get_text() function to create the text you want to use. In this function, you can also choose the part name which you want to fill out. Then, call the view_set_text() function with the part name and text you obtained.

    In this example, the text to use is news_txt and the part name to fill out is widget.text.

    news_txt = data_get_text("widget.text");
view_set_text(wid, "widget.text", news_txt);
  • Image

    You can create a widget with image files using the view_set_image() function. Pass the part name which you want to draw and the name of image file including its directory path.

    In this example, the part name is widget.bg and the name of image file is bg.png in the images directory.

    view_set_image(wid, "widget.bg", "images/bg.png");

Main Operation

The view_set_event_callback() function controls the widget main operation that launches the application.

The launch options are:

  • WIDGET_EVENT_PRESSED for launch on a touch press event
  • WIDGET_EVENT_RELEASED for launch on a touch release event
  • WIDGET_EVENT_CLICKED for launch on a click event

In this example, the application is launched when you click the widget.

view_set_event_callback(wid, "widget.event", WIDGET_EVENT_CLICKED, clicked_cb);
