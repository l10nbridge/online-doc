Dynamic Box

You can add a Dynamic Box to your Web application. The Dynamic Box uses Web technology, such as HTML5, JavaScript, and CSS. Tizen manages the Dynamic Box's life-cycle: installation, creation, updating, termination, and uninstallation.

Note
The Dynamic Box feature is optional for both Tizen mobile and wearable profiles, which means that it may not be supported in all mobile and wearable devices.

To provide a Dynamic Box, define the Dynamic Box basic information in the config.xml file of its Web application using the <tizen:app-widget> tag. If the Web application has the <access> configuration in the config.xml file, its settings are also applied to the Dynamic Box.

The Dynamic Box information management includes:

  • Resource management

    Dynamic Box resources are included in its Web application package. Tizen installs the Dynamic Box when its Web application is installed, and uninstalls the Dynamic Box when its Web application is uninstalled. The Dynamic Box resources are all removed with its Web application.

  • Life-cycle management

    You can create Dynamic Box instances through the Dynamic Box Viewer application (for example, launcher or home application).

    If the creation request is accepted, Tizen starts to render the Dynamic Box content. The onload event is fired, and the box content is alive for 10 seconds. After this all content operations, including painting, are stopped.

    The Dynamic Box can be reloaded and updated at specific situations. The user can request Dynamic Box instances to be terminated through the Dynamic Box Viewer application.

The following main features must be considered during Dynamic Box development:

  • The user can resize the Dynamic Box instance, which results in the Dynamic Box being reloaded. You can define the box content of your Dynamic Box to be fitted to the window size through the viewport setting.
  • The Dynamic Box instance can be reloaded at specific situations. You can define the box content of your Dynamic Box to be proper to such situations by parsing the box URL query string.
  • A Web application can request to reload its Dynamic Box directly or periodically.
  • By default, the box backgrounds are transparent. Therefore, if a body element background color is not set, you can see a part of the Dynamic Box Viewer application behind your Dynamic Box.
  • Some CSS features, such as CSS 2D/3D transforms, CSS transition, CSS animation, CSS opacity, CSS mask, the position: fixed and -webkit-overflow-scrolling: touch attributes, WebGL, and the video element are not supported on the Dynamic Box.
  • Tizen Web device APIs are not supported on the Dynamic Box.
  • By default, the Dynamic Box cannot receive touch events from a user. To receive down and up touch events, the mouse-event attribute in the <tizen:box-content> element must be true.

    Even if the mouse-event attribute is set to true, the mouse move event, including vertical and horizontal swipe, is not supported on the Dynamic Box to guarantee the home screen behavior.
