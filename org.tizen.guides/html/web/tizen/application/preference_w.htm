Preference

Preferences management gives the possibility to store and retrieve applications' preferences. The application developer may create, store and retrieve all preferences needed by his application. They are custom preferencesa Each preference has its own key and value. The value can be a string, number or a boolean value.

The Preference API is mandatory for Tizen mobile profile, which means that it is supported in all mobile devices. All mandatory APIs are supported on the Tizen Emulators.

The main preference manager features are:

  • Setting up a preference.

    A preference can be set by the function setValue(). If the preference exists before setting up, its value is only set. If the preference doesn't exist, it is created.

  • Getting up a preference's value.

    You can get a preference's value in two ways. First you can get a value of a single preference using the function getValue(). If the preference exists, its value is returned. Otherwise an exception is thrown. You can check if the preference exists using the function exists(). The second way is to get values of all preferences. You can get them using the asynchronous function getAll().

  • Removing a preference.

    A preference may be removed by the functions remove() and removeAll(). The first one removes only one preference and the second removes all preferences.

  • Listening for a preference value's change.

    You can provide a listener function what will be called every time a given preference's value changes. You can do this using the function setChangeListener(). The provided listener function will be called with a preference's key and its new value. If you want to put off the listener, you can use the function unsetChangeListener().
