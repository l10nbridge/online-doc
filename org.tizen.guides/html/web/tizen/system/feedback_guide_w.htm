Feedback

Tizen enables you to set system pre-defined media or vibration patterns as a reaction to specified actions.

Feedback API is optional for both Tizen mobile and wearable profiles, which means that it may not be supported in all mobile and wearable devices.

The main feature of the Feedback API is to set (and unset) system pre-defined patterns as a reaction to specified actions.

Each feedback pattern can have separate media files of each types (sound and vibration). For example when email incoming pre-defined sound and/or vibration feedback pattern can be played.

With the isPatternSupported() method, you can determine if for system pre-defined pattern specified media pattern type is available.

Following system pre-defined system patterns are supported

  • TAP - general touch
  • SIP - touch text key
  • KEY0 - touch numeric 0 key
  • KEY1 - touch numeric 1 key
  • KEY2 - touch numeric 2 key
  • KEY3 - touch numeric 3 key
  • KEY4 - touch numeric 4 key
  • KEY5 - touch numeric 5 key
  • KEY6 - touch numeric 6 key
  • KEY7 - touch numeric 7 key
  • KEY8 - touch numeric 8 key
  • KEY9 - touch numeric 9 key
  • KEY_STAR - touch star key
  • KEY_SHARP - touch sharp key
  • KEY_BACK - touch backspace key
  • HOLD - touch hold
  • HW_TAP - press hardware key
  • HW_HOLD - holding press hardware key
  • MESSAGE - incoming a message
  • EMAIL - incoming a email
  • WAKEUP - alert wake up call
  • SCHEDULE - alert schedule alarm
  • TIMER - alert timer
  • GENERAL - alert general event
  • POWERON - power on
  • POWEROFF - power off
  • CHARGERCONN - connecting charger
  • CHARGING_ERROR - occuring charging error
  • FULLCHARGED - full charged
  • LOWBATT - low battery
  • LOCK - lock
  • UNLOCK - unlock
  • VIBRATION_ON - turn on vibration mode
  • SILENT_OFF - turn off silent mode
  • BT_CONNECTED - connecting with bluetooth
  • BT_DISCONNECTED - disconnecting with bluetooth
  • LIST_REORDER - list reorder
  • LIST_SLIDER - list slider sweep
  • VOLUME_KEY - pressed volume key

Each feedback pattern can have separate media files of each types.
