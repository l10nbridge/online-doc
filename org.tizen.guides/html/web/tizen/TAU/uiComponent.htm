UI Component

In this section, we describe multiple-way to manage UI Components.

Defining UI Component

Each UI Component (widget) in TAU has own selector for auto dectect in HTML file. The most popular selectors are classes, old style of selectors is data-role which is deprecated. Some UI Components have also simmle HTML selectors like Button (Button component), input[type=checkbox] (CheckboxRadio component).
If you want detail of each selectors name, please refer UI Component Reference.

With class selector

We recommend to use class selector for each component. Class selectors in TAU are composed with "ui-"prefix and <COMPONENT_NAME>.

Here is HTML example of creating some components with class selector:

<!-- Create Expandable component -->
<div class="ui-expandable" id="expandable-test">
   <h1>Expandable head</h1>
   <div>Content</div>
</div>
<!-- Create ToggleSwitch component -->
<select class="ui-toggleswitch">
   <option value="off"></option>
   <option value="on"></option>
</select>

With data-role selector

data-role selector in TAU composed with <COMPONENT_NAME> in lowercase.

Here is HTML example of creating some components with data-role selector:

<!-- Create TextEnveloper component -->
<div data-role="textenveloper"></div>
<!-- Create Drawer component -->
<div data-role="drawer">
   <ul data-role="listview">
      <li><a href="#">List item 1</a></li>
   </ul>
</div>

Setting options to UI Component

TAU supports several way of setting options for UI Component.
There are several options in each component. For more information, please refer UI Component Reference

Init options with data- attribute

Various options can be set at creating time with data- attribute. This type of setting options is called only when component is created. After firs create of component changeing data attributes on HTML element not change component options.
Here, please refer SectionChanger code with data- option:

<div id="hasSectionchangerPage" class="ui-page">
   <header class="ui-header">
      <h2 class="ui-title">SectionChanger</h2>
   </header>
   <div class="ui-section-changer" data-orientation="horizontal" data-circular="true" data-use-tab="true">
      <div>
         <section>
            <h3>LEFT1 PAGE</h3>
         </section>
         <section class="ui-section-active">
            <h3>MAIN PAGE</h3>
         </section>
         <section>
      </div>
   </div>
</div>

data-circular and data-use-tab will be initial option for creating SectionChanger.

options with Manual Constructor

Options can be also set as argument to component constructor. When using options as argument you should use camelCase name.
Please refer following example. It is same as data- option.:

<div id="hasSectionchangerPage" class="ui-page">
   <header class="ui-header">
      <h2 class="ui-title">SectionChanger</h2>
   </header>
   <div class="ui-section-changer" id="sectionchanger">
      <div>
         <section>
            <h3>LEFT1 PAGE</h3>
         </section>
         <section class="ui-section-active">
            <h3>MAIN PAGE</h3>
         </section>
      </div>
   </div>
</div>
<script>
   var sectionEl = document.getElementById("sectionchanger"),
       sectionChangerWidget = tau.widget.SectionChanger(sectionEl, { 
           orientation: "horizontal",
           circular: true
           useTab: true
       });
</script>

options with method call

To set options dynamically, use option() method.

<div id="hasSectionchangerPage" class="ui-page">
   <header class="ui-header">
      <h2 class="ui-title">SectionChanger</h2>
   </header>
   <div class="ui-section-changer" data-orientation="horizontal" data-circular="true" data-use-tab="true">
      <div>
         <section>
            <h3>LEFT1 PAGE</h3>
         </section>
         <section class="ui-section-active">
            <h3>MAIN PAGE</h3>
         </section>
         <section>
      </div>
   </div>
</div>
<script>
   var sectionEl = document.getElementById("sectionchanger"),
       sectionChangerWidget = tau.widget.SectionChanger(sectionEl);

   sectionChangerWidget.option("circular", true);
</script>

Managing UI Component with JQuery

Developers can use JQuery with TAU for convenience. Also for backward compatibility, TAU supports JQuery interface for UI components. BUT, we strongly recommend to use new TAU style.

Creating UI components

If the jQuery library is loaded, you can use this way to creating component:

<div class="ui-indexscrollbar" id="indexscrollbar"></div>
<script>
   $("#indexscrollbar").indexscrollbar();
</script>

Using methods

If the jQuery library is loaded, you can use this way to call method of component:

$(".selector").componentName("methodName", argument1, argument2, ...);
<div class="ui-indexscrollbar" id="indexscrollbar"></div>
<script>
   // If IndexScrollBar component is created
   $("#indexscrollbar").indexscrollbar("destroy");
</script>
