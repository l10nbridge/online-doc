File Transfer

The File Transfer API gives the possibility to transfer files using HTTP requests. You can download and upload files. You can also track a transfer's progresss and abort it if you need. The default HTTP method is POST, but it is possible to use PUT.

Download needs the following privileges:

  • http://tizen.org/privilege/filesystem.read
  • http://tizen.org/privilege/filesystem.write

Upload needs the following privileges:

  • http://tizen.org/privilege/filesystem.read
  • http://tizen.org/privilege/internet

In order to abort a transfer you need http://tizen.org/privilege/internet privilege.

Note
You need to wait for deviceready event before using File Transfer Cordova API. 
document.addEventListener( "deviceready", onDeviceReady );

function onDeviceReady() {
    //Cordova is ready...
}

All operations (ie. download, upload and abort) are accesible by the FileTransfer object.

Download

To download a file from a server first you must create a FileTransfer object. Then you call its download method. It is asynchronous, so you need to provide callback function which is invoked when the operation is successful. As its argument you get a just downloaded FileEntry object. If you need to track errors you can provide an optional error callback as a argument of download method. See the example.

Upload

The upload operation is quite similiar to download. To upload a file to a server, first you must create a FileTransfer object the same as in download operation. Then you call its upload method providing all necessary data as its arguments. They are URL of an uploaded file, server location and a success callback function, which is invoked when upload is finished successfully. The same as in download can provide an optional error callback as a argument. See the example.

Abort

To abort an in-progress transfer you can use abort method of the FileTransfer object. First to have to create FileTransfer object. Next you start the download or upload transfer operation (as described above) and call the abort method. It stops a transfer and an error FileTransferError.ABORT_ERR is sent to error callback (if it is provided). See the example.

Tracking progress

The FileTransfer object has the property onprogress. It is used to set up a function invoked each time a chunk od data is transferred. As its argument it gets a ProgressEvent object. See the example.
