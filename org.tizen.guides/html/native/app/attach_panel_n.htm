Attach panel

Concept

The attach panel allows the device user to attach various content into an application that contains an attach panel. The user can attach images, take pictures, record voice, and select files on the attach panel.

This feature is supported in mobile applications only.

Figure: Attach panel

Overview

The application developer can create an attach panel to provide the way to attach some files for user.

Also application developer can manage the attach panel.

The attach panel has UI components and manages user interactions on its interface. The layout component keeps on the tabbar and scroller components, which are connected to show the content synchronously. The scroller component has pages to display the content of, for example, images, camera, and voice recorder. Some contents are shown as page on the panel, while others can be launched from the panel's More page by application controls.

The attach panel has half mode and full mode.
The user can change the mode by swiping up and down the page. When the user add the file in full mode, attach panel is hidden automatically. But in half mode, the user can add the file several times.

Figure: Attach panel mode

Contents

Images

To using images content, the attach panel API requires the http://tizen.org/privilege/mediastorage privilege.
In half mode, user can select only one image for attaching image. While in full mode, user can select number of images at once.

Figure: Images content

Camera

To using camera content, the attach panel API requires the http://tizen.org/privilege/mediastorage and http://tizen.org/privilege/cameraprivilege.

Figure: Camera content

Voice

To using voice recorder content, the attach panel API requires the http://tizen.org/privilege/recorderprivilege.

Figure: Voice content

More page

To using More page, the attach panel API requires the http://tizen.org/privilege/appmanager.launchprivilege.
The More page show the icons, for example, video, audio, calendar, contact, myfiles, video recorder, that can be launched by clicking its icon.

Figure: More content
