UI Containers

All applications use containers to create the UI layout. The EFL and Elementary support several ways in which to create this layout: you can use predefined or animated layouts, or even create your own layouts.

Note
Container UI components do not have their own visual identity. They are logical structures used to assign and place other UI components.

When you use container components, such as naviframe, conformant, and box, you must understand certain alignment-related APIs and EDC part positioning for, for example, the core and animated layout. The use of the container UI components only differs from the use of the UI components through their absence of a visual identity.

The following figure shows the basic EFL application structure, and the main container components used in most applications:

  • The conformant container component accounts for the space taken up by the indicator, virtual keyboard, and softkey windows.

    The conformant content is resized and positioned based on the space available. When the virtual keyboard is displayed, the content does not change.

    You always need the conformant component, if your application requires the indicator, virtual keyboard, or softkey windows.

  • The naviframe container component is a top-layer object that allows you to easily manage and switch between multiple views in an application.

    You can first create the naviframe and add the application views (pages) to it, and then use the naviframe to facilitate the navigation between the views.

    If the application has 1 view only, no naviframe component is needed.

Figure: Basic EFL application structure

The following table lists the available UI container components.

Table: Available container components
Container name Description
Box A box container allows you to arrange the UI components in a specific order horizontally or vertically. It is one of the simplest EFL containers.
Conformant A conformant is a container component that accounts for the space taken by the indicator, virtual keyboard, and softkey windows.
Grid In a grid, objects are placed in specific positions along a fixed grid.
Layout A layout is a container component that takes a standard Edje design file and wraps it very thinly in a UI component.
Mapbuf A mapbuf is a container component that uses an Evas map to hold a content object. This UI component is used to improve the moving and resizing performance of complex UI components.
Naviframe A naviframe component consists of a stack of views. New views are pushed on top of previous ones, and only the top-most view is displayed.
Panes
(in mobile applications only)		 A panes component adds a draggable bar between 2 content sections. The sections are resized when the bar is dragged.
Scroller A scroller holds (and clips) a single object and allows you to scroll across it.
Table A table is like a box but with 2 dimensions.
