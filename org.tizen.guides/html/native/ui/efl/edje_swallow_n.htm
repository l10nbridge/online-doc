Creating Edje Swallows

The parent of all layout components is Edje. Edje, as detailed in the EDJE Data Collection Reference, is a library which converts EDC files into Evas objects. One of its main features is the possibility to create SWALLOW type objects. When a swallow object is added in Evas, it can contain any other Evas_Object.

To create a swallow part:

  1. Create the EDC file.

    In this example, the file describes a container group, which contains a SWALLOW type part named part.swallow. The part is centered inside the parent (it is the default behavior) but there are 32 pixels free all around this part.

    collections
{
   group
   {
      name: "container";
      parts
      {
         part
         {
            name: "part.swallow"
            type: SWALLOW;
            description
            {
               state: "default" 0.0;
               rel1.offset: 31 31;
               rel2.offset: -32 -32;
            }
         }
      }
   }
}

    The EDC file is automatically complied into a binary EDJ file when you build the application in the IDE. To manually compile the EDC file separately, use the edje_cc tool:

    edje_cc -o container.edj container.edc
  2. Create an Edje object and load the group from the EDJ file: 
    edje = edje_object_add(evas_object_evas_get(parent));
edje_object_file_set(edje, "container.edj", "container");
    Note
    The edje_object_add() function, as opposed to an elementary object, does not take an Evas_Object as a parent. You must give the function the Evas on which the object is added. As the parent is already added on the Evas by elementary, retrieve a reference to it by using the evas_object_evas_get() function.

    Use the edje_object_file_set() function to set the Edje file from which the object is loaded. The object itself is the name of the group as defined in the EDC file. In this case, container.

  3. Use the edje_object_part_swallow() function to swallow any kind of Evas_Object inside the Edje object.

    In this example, an icon object is swallowed.

    ic = elm_icon_add(parent);
elm_image_file_set(ic, "c1.png", NULL);
edje_object_part_swallow(edje, "part.swallow", ic);
    Note
    The elm_image_file_set() function parameters are linked to Edje. The second parameter in this example is a PNG file, but it can also be an Edje file. In that case, the third parameter must be the Edje group to load, exactly as previously shown with the edje_object_file_set() function.
